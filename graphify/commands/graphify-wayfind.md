---
name: graphify-wayfind
description: Generate a breadcrumb path from the vault MOC root to the given topic, via graphify's connect_topics shortest-path. Stages the breadcrumb as a pending vault note — user must run `graphify approve <id>` before anything is written.
argument-hint: <topic>
disable-model-invocation: true
target: obsidian
---

Arguments: $ARGUMENTS

Treat `$ARGUMENTS` as the target topic — the destination node label or node id.

**Step 1 — resolve the MOC root (D-05 heuristic: largest community, tie-break lowest community_id):**

Call the graphify MCP tool `get_community` with:
- `community_id`: 0

Communities are re-indexed after clustering so that community 0 is the largest (see `graphify/cluster.py`). If the response `meta.status` is `no_graph`, render:
> No graph found at graphify-out/graph.json. Run `/graphify` to build one, then re-invoke `/graphify-wayfind`.
and stop.

If `meta.status` is `community_not_found`, the graph is empty or degenerate — render:
> Graph has no clustered communities. `/graphify-wayfind` needs at least one community to pick a MOC root. Rebuild the graph on a non-empty project.
and stop.

From the `get_community` response, take `meta.top_labels[0]` as the MOC root label. Call this `moc_root_label`.

**Step 2 — shortest path from MOC root to target:**

Call the graphify MCP tool `connect_topics` with:
- `topic_a`: `<moc_root_label>`
- `topic_b`: `$ARGUMENTS`
- `budget`: 500

Parse the D-02 envelope's `meta.status`.

**If `status` is `entity_not_found`:** render:
> The target topic `$ARGUMENTS` is not in the current graph. Try a different term or rebuild the graph.
and stop.

**If `status` is `no_path`:** render:
> No path exists from the MOC root (`<moc_root_label>`) to `$ARGUMENTS` in this graph. The target may be in a disconnected component.
and stop.

**If `status` is `ok`:** proceed. Extract the shortest-path chain of labels from the response `text_body`.

**Step 3 — render the breadcrumb and stage via `propose_vault_note`:**

Render the breadcrumb as a Markdown body:

```
# Wayfinding to $ARGUMENTS

`[[<moc_root_label>]]` → `[[<hop-1>]]` → `[[<hop-2>]]` → … → `[[$ARGUMENTS]]`

## Full chain

- [[<moc_root_label>]]
- [[<hop-1>]]
- [[<hop-2>]]
- …
- [[$ARGUMENTS]]

## Source

Generated by `/graphify-wayfind` using `connect_topics` shortest-path.
```

Then call the graphify MCP tool `propose_vault_note` with:
- `title`: `Wayfind - $ARGUMENTS`
- `body_markdown`: <rendered body above>
- `suggested_folder`: profile.folder_mapping.wayfind (fall back to `wayfind/` if absent)
- `note_type`: `"wayfind"`
- `rationale`: `"Phase 14 /graphify-wayfind from <moc_root_label> to $ARGUMENTS"`

Tell the user the proposal ID and instruct them to run `graphify approve <id>` to commit the breadcrumb to their vault — do NOT write to the vault yourself.
