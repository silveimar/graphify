## Cross-Phase Integration Issues

### INT-01 (High): Approve flow bypasses vault manifest

**Location:** `graphify/__main__.py` lines 834-835
**Affected:** TRIP-01, TRIP-02, MCP-07

`_approve_and_write_proposal` calls `compute_merge_plan` and `apply_merge_plan` without passing `manifest=` or `manifest_path=` parameters. This means:

1. **User-modified detection bypassed** — If a target note was edited by the user since the last `--obsidian` run, the approve flow will not detect this and may overwrite the note (TRIP-02 not enforced on this path).
2. **Manifest not updated** — After writing the approved note, `vault-manifest.json` is not updated with the new content hash. A subsequent `--obsidian` re-run may treat the approved note as unmanaged (TRIP-01 not enforced on this path).
3. **Sentinel blocks still preserved** — `_synthesize_file_text` unconditionally extracts and restores user sentinel blocks, so TRIP-04/TRIP-06 hold even without manifest threading.

**Fix:** Load manifest before `compute_merge_plan` in `_approve_and_write_proposal`; pass `manifest_path` and `old_manifest` to `apply_merge_plan`.

### INT-02 (Medium): Skill orchestrator missing auto_snapshot_and_delta

**Location:** `graphify/skill.md` Step 4 / Step 9
**Affected:** DELTA-01, DELTA-02

`auto_snapshot_and_delta` is exported from `snapshot.py`, registered in `__init__.py`, and wired to the `graphify snapshot` CLI subcommand. However, `skill.md` (the primary `/graphify` invocation path) never calls it. Users running the skill get no automatic snapshot or delta report — they must manually run `graphify snapshot --delta`.

**Fix:** Add `auto_snapshot_and_delta(G, communities)` call after `to_json` in skill.md Step 4 or Step 9.

### INT-03 (Low): MCP get_node missing provenance fields

**Location:** `graphify/serve.py` lines 520-528
**Affected:** DELTA-04

`_tool_get_node` returns label, ID, source, type, community, and degree but not `extracted_at`, `source_hash`, or staleness classification. Agents cannot query a node's freshness through MCP.

**Fix:** Extend `_tool_get_node` response to include provenance fields; optionally expose `classify_staleness` as an MCP tool.
