# Requirements: graphify — Milestone v1.4 Agent Discoverability & Obsidian Workflows

**Defined:** 2026-04-17
**Scope confirmed:** P1 + P2 complete (no stretch deferrals beyond documented v1.4.x carve-outs)
**Core Value:** Turn graphify from a tool you run into a tool that agents find, trust, compose with, and improve in the background. Agent discoverability via MCP capability manifest + harness-memory export; Obsidian workflow depth via vault-scoped thinking commands; graph-quality over time via heterogeneous routing, async enrichment, focus-aware zoom, grounded chat, and SPAR-Kit graph-argumentation — all layered on v1.0–v1.3 invariants (graph.json read-only, atomic sidecars, `peer_id="anonymous"`, `security.py` gate).

**Priority order (locked):** Phase 12 → 18 (parallel) → 13 Wave A → 15 → 17 → 16 → 14 (with Plan 00 whitelist refactor) → 13 Wave B + SEED-002. See `.planning/research/SUMMARY.md` for the edge-level dependency graph.

**Research synthesis:** `.planning/research/SUMMARY.md` — 4-dimension synthesis (Stack / Features / Architecture / Pitfalls) with reconciled build order, top-8 pitfalls, and v1.0–v1.3 regression-risk table v1.4 must preserve.

## v1.4 Requirements

Requirements grouped by REQ-ID prefix, each mapping to exactly one roadmap phase. `[P2]` items are confirmed in scope for v1.4 per user decision on 2026-04-17; `[stretch]` would be the capacity-pressure drop list (none tagged as stretch in v1.4 — P2 is the new stretch boundary).

### Heterogeneous Extraction Routing (Phase 12)

- [ ] **ROUTE-01**: A new `graphify/routing.py` module classifies each file into a complexity class (`trivial` / `simple` / `complex`) using AST-derived metrics (cyclomatic complexity via `radon` for Python, tree-sitter-walker metrics for other languages: nesting depth, import count, symbol count)
- [ ] **ROUTE-02**: Router selects an extraction model per complexity class from a `graphify/routing_models.yaml` config file so users can retarget without code edits (mirrors `harness_schemas/` pattern)
- [ ] **ROUTE-03**: `extract()` supports an optional `router: Router | None = None` kwarg; when provided, per-file extractor dispatch goes through the router and LLM calls fan out via `concurrent.futures.ThreadPoolExecutor` (default 4 workers, `GRAPHIFY_EXTRACT_WORKERS` env override)
- [ ] **ROUTE-04**: `cache.py::file_hash()` gains an optional `model_id: str = ""` parameter so cache keys are `SHA256(bytes) + ":" + model_id` — prevents cross-contamination between routing decisions while staying backward compatible with callers that pass no `model_id`
- [ ] **ROUTE-05**: A `graphify-out/routing.json` sidecar records `{file_path → {class, model, endpoint, tokens_used, ms}}` per run with atomic `.tmp` + `os.replace` write
- [ ] **ROUTE-06**: A hard floor per `file_type` prevents silent quality regression — code files always route to at least the mid-tier model even if AST metrics classify them as trivial (Pitfall 1 mitigation)
- [ ] **ROUTE-07**: Parallelism is stampede-safe — a central `threading.Semaphore` bounds provider concurrency and a shared `threading.Event` implements global 429 backoff so N parallel retries can't thundering-herd (Pitfall 2 mitigation)
- [ ] **ROUTE-08** `[P2]`: Canary probes re-run every Nth cheap-routed file on the expensive model with an edge-count-ratio ≥ 0.6 guard; failures raise a user-facing "routing quality regressed" warning
- [ ] **ROUTE-09** `[P2]`: `GRAPHIFY_COST_CEILING` env var enforces pre-flight cost projection; extraction aborts with a clear message when the projected budget would be exceeded
- [ ] **ROUTE-10** `[P2]`: Image file types route to a vision-capable model slot (separate from the 3 text classes); absent model in `routing_models.yaml` falls back to skipping images (current behavior)

### Agent Capability Manifest (Phase 13)

- [ ] **MANIFEST-01**: A static `server.json` at repo root complies with MCP spec 2025-11-25 (servers-registry format) so external registries can auto-discover graphify
- [ ] **MANIFEST-02**: A runtime `graphify-out/manifest.json` regenerates on every full pipeline run so the manifest reflects the currently-installed graphify version
- [ ] **MANIFEST-03**: New MCP tool `capability_describe` returns the static manifest merged with live state (graph stats, alias map size, sidecar freshness, enrichment snapshot id)
- [ ] **MANIFEST-04**: New CLI subcommand `graphify capability [--validate] [--stdout]` dumps JSON to stdout and validates against the published schema (JSON Schema draft 2020-12 via `jsonschema` — transitive via `[mcp]`, no new extra)
- [ ] **MANIFEST-05**: Manifest is generated by introspection over `serve.py::list_tools()`, `_handlers`, and CLI dispatch — **never hand-maintained** (Pitfall 12 mitigation); a `CAPABILITY_TOOLS` top-level list is the single source of truth
- [ ] **MANIFEST-06**: Each tool entry declares `cost_class: free|cheap|expensive`, `deterministic: bool`, `cacheable_until`, `composable_from: [tool_name]` so agents can reason about cost + composition before calling
- [ ] **MANIFEST-07**: A manifest content-hash is embedded in every MCP response envelope meta so agents detect drift between what was advertised and what responded (Pitfall 13 mitigation)
- [ ] **MANIFEST-08**: Skill frontmatter gains a `capability_manifest: graphify-out/manifest.json` line so other skills can discover graphify's surface declaratively
- [x] **MANIFEST-09** `[P2]`: CI gate `graphify capability --validate` fails builds when manifest drifts from `serve.py` registrations
- [x] **MANIFEST-10** `[P2]`: Per-tool usage examples auto-extracted from docstrings and embedded under `_meta.examples` (extends MCP `_meta` publisher-provided metadata namespace)

### Async Background Enrichment (Phase 15)

- [ ] **ENRICH-01**: New module `graphify/enrich.py` + CLI subcommand `graphify enrich [--pass description|patterns|community|staleness] [--budget TOKENS] [--graph PATH]` (default runs all 4 passes)
- [x] **ENRICH-02**: Four built-in derivation passes — (a) `description` enriches node descriptions from docstrings/comments, (b) `patterns` detects emerging cross-snapshot patterns, (c) `community` generates per-community natural-language summaries, (d) `staleness` refreshes FRESH/STALE/GHOST overrides
- [x] **ENRICH-03**: Enrichment writes **overlay sidecar only** (`graphify-out/enrichment.json`) — `graph.json` is never mutated (v1.1 invariant preserved; Pitfall 3 critical mitigation)
- [ ] **ENRICH-04**: Atomic `.tmp` + `os.replace` write on `enrichment.json`; a shared `fcntl.flock` on `.enrichment.lock` coordinates with foreground `/graphify` to prevent write races
- [x] **ENRICH-05**: Enrichment pins `--snapshot-id` at process start and runs against that snapshot for the whole run — does NOT follow new snapshots mid-run (determinism + race-free delta machinery)
- [x] **ENRICH-06**: Event-driven trigger via `watch.py` post-rebuild hook (opt-in `--enrich` flag); **no `apscheduler` dependency** per OQ-1 resolution — manual + event-driven is sufficient for v1.4
- [x] **ENRICH-07**: Foreground `/graphify` rebuild grabs the `.enrichment.lock` and enrichment SIGTERM-aborts cleanly without corrupting partial writes
- [x] **ENRICH-08**: `serve.py::_load_enrichment_overlay(out_dir)` merges enrichment at read time (post-load, not inside `_load_graph`) so tests mocking `_load_graph` remain unaffected
- [x] **ENRICH-09**: `serve.py::_reload_if_stale()` adds mtime watch for `enrichment.json` alongside existing `graph.json` watch
- [x] **ENRICH-10** `[P2]`: `graphify enrich --dry-run` emits cost preview (estimated tokens per pass, per-node budget breakdown) without calling LLMs
- [x] **ENRICH-11** `[P2]`: Soft-dependency on Phase 12 `routing.json` — description pass skips files already extracted by expensive model (no double-LLM cost)
- [x] **ENRICH-12** `[P2]`: Alias redirect (D-16) threaded through enrichment writes — derived content keys by canonical node_id

### Graph Argumentation Mode (Phase 16)

- [x] **ARGUE-01**: New module `graphify/argue.py` — substrate only. Exposes `populate(G, topic|subgraph, *, scope: "topic"|"subgraph"|"community", budget: int) -> ArgumentPackage`
- [x] **ARGUE-02**: `ArgumentPackage` dataclass fields: `subgraph: nx.Graph`, `perspectives: list[PerspectiveSeed]`, `evidence: list[NodeCitation]`
- [x] **ARGUE-03**: LLM orchestration lives in `skill.md` (D-73 honored); `argue.py` contains zero LLM calls — parallels Phase 9 autoreason tournament structure
- [x] **ARGUE-04**: New MCP tool `argue_topic(topic, scope, budget)` returns `ArgumentPackage` serialized into D-02 meta envelope
- [x] **ARGUE-05**: Mandatory `{claim, cites: [node_id]}` schema on every persona turn; validator rejects unknown cites as `[FABRICATED]` and re-prompts (Pitfall 4 mitigation)
- [x] **ARGUE-06**: Phase 9 blind-label harness reused as-is — shuffled A/B labels, stripped persona phrases, rotating judge identity (prevents Defender bias regression)
- [x] **ARGUE-07**: Manifest declares `composable_from: []` for `argue_topic` — Phase 16 MUST NOT invoke Phase 17 chat as primitive (Pitfall 18 recursion guard)
- [x] **ARGUE-08**: Round cap = 6; debate temperature ≤ 0.4; `dissent` and `inconclusive` are valid outputs (no consensus-forcing)
- [x] **ARGUE-09**: Output written to `graphify-out/GRAPH_ARGUMENT.md`; argumentation is **advisory only** — never auto-applies changes to code or graph
- [x] **ARGUE-10**: `/graphify-argue <question>` command file invokes `argue_topic` via MCP
- [x] **ARGUE-11** `[P2]`: SPAR-Kit INTERROGATE step between ABSTRACT and RUMBLE (+40% synthesis quality per protocol docs); opt-in via `--interrogate` flag
- [x] **ARGUE-12** `[P2]`: Persona memory across rounds (persistent perspective identity) — each persona retains its prior claims across the debate to avoid contradiction
- [x] **ARGUE-13** `[P2]`: Clash/rumble/domain intensity scoring annotates the transcript with conflict density metrics for post-hoc review

### Conversational Graph Chat (Phase 17)

- [x] **CHAT-01**: New MCP tool `chat(query: str, session_id: str | None = None)` runs a two-stage pipeline structurally enforced (Stage 1 tool-call only → Stage 2 compose from results only)
- [x] **CHAT-02**: Stage 1 translates NL query to a tool-call sequence over `_score_nodes` / `_bfs` / `_dfs` / `_bidirectional_bfs` / `_get_community` / `_connect_topics`
- [x] **CHAT-03**: Stage 2 composes a structured narrative packet from the tool results; `serve.py` makes zero LLM calls (skill or calling agent renders the human-facing narrative)
- [x] **CHAT-04**: Every claim paired with `{node_id, label, source_file}` in `meta["citations"]`; empty or fabricated cites rejected by post-process grep (Pitfall 5 mitigation)
- [x] **CHAT-05**: Empty graph-query results return templated fuzzy suggestions ("did you mean X, Y?") — NEVER fabricate node names
- [x] **CHAT-06**: `/graphify-ask <question>` command file in `graphify/commands/` (target: both) invokes `chat` MCP tool
- [x] **CHAT-07**: D-16 alias redirect threaded through `_run_chat` handler so chat-cited node_ids resolve to canonical after dedup
- [x] **CHAT-08**: Session history scoped per `session_id`; no cross-session memory (privacy by default)
- [x] **CHAT-09**: D-02 MCP envelope + response size cap (≤ 500 tokens narrative, meta envelope unlimited)
- [ ] **CHAT-10** `[P2]`: Auto-suggest follow-up questions derived from surprising-connections neighbors of the answered subgraph

## Out of Scope

- **Real-time cursor-position tracking** (Phase 18) — LSP integration is a separate project
- **Ambient filesystem watcher for focus** (Phase 18) — contradicts stateless-per-call MCP design
- **Persistent cross-session focus** (Phase 18) — privacy: focus is per-query only
- **Multi-file focus** (Phase 18) — single-file focus composes via repeated calls
- **Obsidian plugin development** (Phase 14) — output is standard markdown + frontmatter; plugin is user-side
- **File-watcher-triggered Obsidian commands** (Phase 14) — explicit user invocation only
- **Real-time enrichment rebuild** (Phase 15) — contradicts Phase 6 delta machinery
- **Blocking-pipeline enrichment** (Phase 15) — enrichment runs async, never blocks foreground
- **Arbitrary user-Python derivers** (Phase 15) — entry-point plugin model deferred; 4 built-in passes only
- **Implicit enrichment on MCP `get_node`** (Phase 15) — no surprise LLM calls in query path
- **LLM budget-less enrichment** (Phase 15) — all passes must accept `--budget` cap
- **Cross-session chat memory** (Phase 17) — privacy; session-scoped only
- **Chat without tool call** (Phase 17) — structurally enforced two-stage pipeline
- **Chat answers > 500 tokens** (Phase 17) — escape valve: full result lives in meta envelope
- **Phase 16 invoking Phase 17 chat** — forbidden composition (Pitfall 18)
- **Phase 16 auto-applying recommendations** — argumentation is advisory-only
- **Argumentation round cap > 6** — cost + bias risk escalates past this point
- **Consensus-forcing debate outputs** — `dissent` / `inconclusive` are valid
- **Router learning from feedback** (Phase 12) — hand-tuned thresholds + manual per-file-type table only
- **Per-function routing within a single file** (Phase 12) — breaks extraction atomicity
- **Silent cache fallback when expensive model fails** (Phase 12) — surfaces errors instead
- **20+ model auto-discovery** (Phase 12) — 3 classes + 1 vision slot; static config
- **Proprietary manifest schema** (Phase 13) — extends MCP `server.json._meta`; no fork
- **Auth in manifest** (Phase 13) — MCP transport layer handles auth
- **Dynamic tool-appearance (tools registering at runtime)** (Phase 13) — static introspection at start
- **Auto-publish to MCP registry** (Phase 13) — user runs `graphify capability` manually
- **Annotation export by default** (SEED-002) — allow-list or `--include-annotations` opt-in
- **MCP-call-trace export** (SEED-002) — privacy: never export telemetry envelopes
- **Auto-export on every build** (SEED-002) — explicit user invocation
