### `_bidirectional_bfs()` (NEW — traversal algorithm, request-response)

**Analog:** `graphify/serve.py:350` — `_bfs()` (single-endpoint unidirectional)

**Contract to mirror** (lines 350–364):

```python
def _bfs(G: nx.Graph, start_nodes: list[str], depth: int) -> tuple[set[str], list[tuple]]:
    visited: set[str] = set(start_nodes)
    frontier = set(start_nodes)
    edges_seen: list[tuple] = []
    for _ in range(depth):
        next_frontier: set[str] = set()
        for n in frontier:
            for neighbor in G.neighbors(n):
                if neighbor not in visited:
                    next_frontier.add(neighbor)
                    edges_seen.append((n, neighbor))
        visited.update(next_frontier)
        frontier = next_frontier
    return visited, edges_seen
```

**How new code diverges from this analog** (per RESEARCH §"Pattern 2" lines 242–319 + CONTEXT D-06/D-07):

1. **Return tuple extended to 3-elem:** `(visited: set[str], edges_seen: list[tuple], status: str)` where `status ∈ {"ok", "frontiers_disjoint", "budget_exhausted"}`. Callers that expect 2-tuple (existing `_bfs()` callers) are unaffected because they call `_bfs()` — not `_bidirectional_bfs()`.
2. **Two frontiers, not one:** maintain `forward_visited`/`forward_frontier` and `reverse_visited`/`reverse_frontier`. Expand the smaller frontier first on each iteration (NetworkX heuristic — RESEARCH line 286 "same as NetworkX's private helper").
3. **Meet-point check every iteration:** `if forward_visited & reverse_visited: return ..., "ok"`.
4. **Depth budget split:** `max_forward = (depth + 1) // 2`; `max_reverse = depth // 2` (RESEARCH lines 276–277).
5. **Visit cap for D-07 `budget_exhausted`:** new parameter `max_visited: int`; check `len(forward_visited) + len(reverse_visited) >= max_visited` inside the loop.
6. **No partial-path reconstruction:** unlike NetworkX's private `_bidirectional_pred_succ`, we do NOT emit pred/succ dicts — we return the union visited set, which `_subgraph_to_text()` already consumes.
7. **`from __future__ import annotations` + type hints** per CLAUDE.md Code Style (already at `serve.py:2`).

**Placement:** Insert between existing `_bfs()` (L350) and `_dfs()` (L366) — matches RESEARCH line 250 "placed between `_bfs()` (L350) and `_dfs()` (L366)".

**Telemetry coupling** (Pitfall 6, RESEARCH lines 67–70): dedupe `edges_seen` before passing to `_record_traversal()` via `list({(min(u,v), max(u,v)) for u,v in edges_seen})` — `_record_traversal()` already normalizes keys but a second counter increment would inflate hot-path detection.

**Also note:** RESEARCH §"Pattern 2" provides the full 60-line reference implementation (lines 253–317). Planner should copy that implementation verbatim as the Wave-0 starting point.

```python
def _estimate_cardinality(
    G: nx.Graph,
    start_nodes: list[str],
    depth: int,
    layer: int,
    branching_factor: float,
) -> dict:
    """1-hop BFS probe + geometric extrapolation. Returns {"nodes","edges","tokens"}."""
    if depth <= 0:
        return {"nodes": len(start_nodes), "edges": 0,
                "tokens": _estimate_tokens_for_layer(len(start_nodes), 0, layer)}
    one_hop_visited, one_hop_edges = _bfs(G, start_nodes, 1)
    one_hop_count = len(one_hop_visited)
    if depth == 1:
        est_nodes = one_hop_count
    else:
        est_nodes = min(
            int(len(start_nodes) * (branching_factor ** depth)),
            G.number_of_nodes(),
        )
    est_edges = min(int(est_nodes * branching_factor / 2), G.number_of_edges())
    return {"nodes": est_nodes, "edges": est_edges,
            "tokens": _estimate_tokens_for_layer(est_nodes, est_edges, layer)}
```

**Divergence from `_check_derived_edges()` analog:** `_estimate_cardinality` returns a dict (not None), reads no sidecar state (pure function of `G` + params + cached branching factor). It is a PURE function — easier to unit-test than `_check_derived_edges`.

```python
def _record_traversal(telemetry: dict, edges: list[tuple]) -> None:
    """Increment traversal counters for each edge. Keys normalized as min:max. Per D-02."""
    counters = telemetry.setdefault("counters", {})
    for u, v in edges:
        key = f"{min(u, v)}:{max(u, v)}"
```

**Reference implementation** (RESEARCH §"Pattern 3" lines 329–364):

```python
def _encode_continuation(
    query_params: dict,
    visited: set[str],
    current_layer: int,
    graph_mtime: float,
) -> str:
    """Encode opaque drill-down token. Base64(JSON). Integrity: sha256 of mtime+params."""
    import base64, hashlib
    mtime_hash = hashlib.sha256(
        f"{graph_mtime}:{json.dumps(query_params, sort_keys=True)}".encode()
    ).hexdigest()[:16]
    payload = {
        "q": query_params,
        "v": sorted(visited),        # deterministic — reproducible in tests
        "l": current_layer,
        "h": mtime_hash,
    }
    return base64.urlsafe_b64encode(json.dumps(payload, sort_keys=True).encode()).decode()


def _decode_continuation(token: str, graph_mtime: float) -> tuple[dict, str]:
    """Decode + validate. Returns (payload, status) where status ∈ {"ok","graph_changed","malformed"}."""
    import base64, hashlib
    try:
        payload = json.loads(base64.urlsafe_b64decode(token.encode()).decode())
    except (ValueError, json.JSONDecodeError):
        return {}, "malformed"
    expected = hashlib.sha256(
        f"{graph_mtime}:{json.dumps(payload.get('q', {}), sort_keys=True)}".encode()
    ).hexdigest()[:16]
    if payload.get("h") != expected:
        return payload, "graph_changed"
    return payload, "ok"
```

**Security notes** (RESEARCH §"Security Domain" lines 267–279):
- Cap raw token length at 64 KB before decode (DoS mitigation).
- Wrap both base64-decode and json-decode in try/except.
- Truncate sha256 hex to 16 chars — collision-resistant for mtime-dedup purposes; do NOT invent a new hash construction.
- No peer_id in payload — drill-down is peer-neutral per D-03 + CONTEXT code_context line "peer_id defaults `'anonymous'`".

### Extend `_subgraph_to_text()` with `layer: int` (IN-PLACE signature extension)

**Analog:** itself — `graphify/serve.py:382–398`:

```python
def _subgraph_to_text(G: nx.Graph, nodes: set[str], edges: list[tuple], token_budget: int = 2000) -> str:
    """Render subgraph as text, cutting at token_budget (approx 3 chars/token)."""
    char_budget = token_budget * 3
    lines = []
    for nid in sorted(nodes, key=lambda n: G.degree(n), reverse=True):
        d = G.nodes[nid]
        line = f"NODE {sanitize_label(d.get('label', nid))} [src={d.get('source_file', '')} loc={d.get('source_location', '')} community={d.get('community', '')}]"
        lines.append(line)
    for u, v in edges:
        if u in nodes and v in nodes:
            d = G.edges[u, v]
            line = f"EDGE {sanitize_label(G.nodes[u].get('label', u))} --{d.get('relation', '')} [{d.get('confidence', '')}]--> {sanitize_label(G.nodes[v].get('label', v))}"
            lines.append(line)
    output = "\n".join(lines)
    if len(output) > char_budget:
        output = output[:char_budget] + f"\n... (truncated to ~{token_budget} token budget)"
    return output
```

**Divergence plan:**
- Add `layer: int = 3` parameter with default `3` (preserves legacy behavior — L3 is "full" rendering, matching the existing output).
- Switch on `layer`:
  - **L1:** emit only `id`, sanitized `label`, `community` — drop `src`, `loc`, relation details. Target ≈ 50 tok/node.
  - **L2:** emit L1 fields + 1-hop neighbor labels + edge `relation` + `confidence`. Target ≈ 200 tok/node + 30 tok/edge.
  - **L3:** current behavior unchanged (full attributes). Calibrated ≈ 100 tok/node + 95 tok/edge.
- **Preserve** `sanitize_label()` on every label emission in every layer path (CLAUDE.md §Security + RESEARCH §"Anti-Patterns to Avoid" bullet 6).
- **Preserve** degree-descending sort at the top (CONTEXT-neutral — existing behavior; good UX property).

**Backward compatibility:** Callers that pass no `layer=` (e.g., old tests, legacy `_tool_query_graph` path) default to L3 and get exactly the current output.

### Extend `_tool_query_graph()` dispatch (control-flow extension)

**Analog:** `graphify/serve.py:631–648`:

```python
def _tool_query_graph(arguments: dict) -> str:
    _reload_if_stale()
    question = arguments["question"]
    mode = arguments.get("mode", "bfs")
    depth = min(int(arguments.get("depth", 3)), 6)
    budget = int(arguments.get("token_budget", 2000))
    terms = [t.lower() for t in question.split() if len(t) > 2]
    scored = _score_nodes(G, terms)
    start_nodes = [nid for _, nid in scored[:3]]
    if not start_nodes:
        return "No matching nodes found."
    nodes, edges = _dfs(G, start_nodes, depth) if mode == "dfs" else _bfs(G, start_nodes, depth)
    _record_traversal(_telemetry, edges)
    _save_telemetry(_out_dir, _telemetry)
    _check_derived_edges(G, _telemetry, _out_dir, _agent_edges)
    header = f"Traversal: {mode.upper()} depth={depth} | Start: {[G.nodes[n].get('label', n) for n in start_nodes]} | {len(nodes)} nodes found\n\n"
    return header + _subgraph_to_text(G, nodes, edges, budget)
```

```python
        G.add_node(f"n{i}", label=f"node{i}", community=0)
    for i in range(1, n):
        G.add_edge(f"n{i}", f"n{i+1}", relation="next", confidence="EXTRACTED")
    return G


def _make_disjoint_components() -> nx.Graph:
    """Two disjoint 3-node chains — for frontiers_disjoint tests (D-07)."""
    G = nx.Graph()
    for i in range(1, 4):
        G.add_node(f"a{i}", community=0)
        G.add_node(f"b{i}", community=1)
    G.add_edge("a1", "a2"); G.add_edge("a2", "a3")
    G.add_edge("b1", "b2"); G.add_edge("b2", "b3")
    return G
```

## Shared Patterns

### `from __future__ import annotations` + PEP 604 union types
**Source:** `graphify/serve.py:2` — `from __future__ import annotations`
**Apply to:** all new code. Use `str | None`, `dict[str, int]`, `tuple[set[str], list[tuple], str]` — NOT `Optional[str]`, `Dict[...]`, `Tuple[...]`. Per CLAUDE.md Code Style.

### `sanitize_label()` on every emitted label
**Source:** `graphify/serve.py:11` (import) and L386 / L390 (usage inside `_subgraph_to_text`)
```python
from graphify.security import sanitize_label
...
line = f"NODE {sanitize_label(d.get('label', nid))} [src=... community=...]"
```
**Apply to:** every new layer-rendering path (L1 / L2 / L3 branches), continuation-token payload emission (labels stored in `visited` set are NOT rendered to the agent until `_subgraph_to_text` runs, but if you emit raw labels anywhere new, wrap in `sanitize_label`). Per CLAUDE.md Security + RESEARCH §Anti-Patterns #6.

### `_reload_if_stale()` as the first line of every tool handler
**Source:** `graphify/serve.py:619–629` — called at the top of `_tool_query_graph` (L632), `_tool_get_node`, etc.
```python
def _reload_if_stale() -> None:
    """Reload G and communities if graph.json mtime has changed (D-13)."""
    nonlocal G, communities, _graph_mtime
    try:
        mtime = os.stat(graph_path).st_mtime
    except OSError:
        return
    if mtime != _graph_mtime:
        G = _load_graph(graph_path)
        communities = _communities_from_graph(G)
        _graph_mtime = mtime
```
**Apply to:** `_tool_query_graph` extension — call BEFORE reading `_branching_factor` (also extend `_reload_if_stale` to recompute `_branching_factor` per D-04). Continuation-token mtime check in `_decode_continuation` reads the refreshed `_graph_mtime` via closure.

**Source:** `graphify/serve.py:325, 328` (in `_load_graph`):

**Apply to:** new error paths — warnings/info go to stderr with `[graphify]` prefix per CLAUDE.md. DO NOT introduce custom exception hierarchy for this phase (CLAUDE.md error style).

### Closure-var module state pattern
**Source:** `graphify/serve.py:447–454` — `G, communities, _graph_mtime, _out_dir, _annotations, _agent_edges, _telemetry, _session_id` all live as closures inside `serve()`.
**Apply to:** new `_branching_factor` — same pattern, same scope, same `nonlocal` declaration in `_reload_if_stale`.

### One-test-per-module convention
**Source:** CLAUDE.md §Testing Conventions: "One test file per module (`tests/test_<module>.py`)"
**Apply to:** ALL new Phase 9.2 tests go in `tests/test_serve.py`. Do NOT create `test_progressive.py`, `test_bidirectional.py`, or `test_continuation.py`. RESEARCH line 204 reiterates this.

### `tmp_path`-only for filesystem side effects
**Source:** `tests/test_serve.py:151` — `test_load_graph_roundtrip(tmp_path)` pattern
**Apply to:** continuation-token tests (in-memory only — no filesystem needed), cardinality tests (in-memory NetworkX graph), bi-BFS tests (synthetic chain/disjoint builders). **Apply to:** new `_make_chain_graph(n)` and `_make_disjoint_components()` — sorted IDs, deterministic edges. Continuation-token tests use `sorted(visited)` (RESEARCH line 343) so payload encoding is reproducible.

**Source:** CLAUDE.md + CONTEXT.md Constraints + RESEARCH §Standard Stack.
**Apply to:** everything. `base64`, `hashlib`, `json` are stdlib. NetworkX already core. No tiktoken, no pydantic, no marshmallow, no new MCP content-type packages.
