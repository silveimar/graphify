## Decisions Made

- **Branching factor empty-graph sentinel = 1.0** — returning 0.0 would make `branching_factor ** depth` collapse to 0, producing a misleading "will yield 0 nodes" estimate. 1.0 is finite and caller-safe (plan implicitly required "no crash"; this is the cleanest way).
- **Task commits not split into RED/GREEN** — plan STEP structure interleaves tests and implementation in each task; splitting would require 6 commits instead of 3 and violate the "one task = one commit" protocol in `<task_commit_protocol>`. RED/GREEN cycles were still run manually before each commit (verified failing import, then verified passing).
- **Decoder returns payload on `graph_changed`** — spec explicitly says "Payload is still returned (not empty) — agent can inspect it for debugging." Kept exactly as specified.
- **Test file `import base64`** — added to test module imports so the tampered-hash test can construct a manipulated token without reaching into the codec internals.
