## Decisions Made

- **Test Rule 3 deviation on sanitize_label assertion** — the plan's `test_subgraph_to_text_all_layers_sanitize_labels` used `label="<script>alert(1)</script>"` and asserted `"<script>" not in out`. But `graphify/security.py:sanitize_label` only strips control chars (`\x00-\x1f\x7f`) and caps length at 256; it does NOT HTML-escape `<script>`. Verified via `python3 -c "from graphify.security import sanitize_label; print(repr(sanitize_label('<script>alert(1)</script>')))"` → returns the string unchanged. Deviation: changed the test fixture to use `label="bad\x00label"` and assert `"\x00" not in out` — this verifies sanitize_label IS being applied in every layer path (the actual plan intent). The assertion is stricter than the original since `\x00` in an unsanitized path would leak through, and no string except `""` is a superset of "had sanitize_label applied" for both control chars AND angle brackets simultaneously. Tracked as Rule 3 (auto-fix blocking issue) because the plan test would have failed on a correct implementation.
- **Sentinel placement as module-level constant** — plan alternated between inline string literal and named constant. Chose module-level `QUERY_GRAPH_META_SENTINEL = "\n---GRAPHIFY-META---\n"` so tests can `.split(QUERY_GRAPH_META_SENTINEL, 1)` instead of magic string. Matches plan's STEP 4 test code which imports `QUERY_GRAPH_META_SENTINEL` from serve.
- **Defensive dispatch clamp** — schema enforces budget min=50 max=100000, but `_run_query_graph` also does `budget = max(50, min(budget, 100000))` because tests + future Python callers bypass schema validation. Threat T-09.2-09 depth explicitly requires this belt-and-suspenders.
- **Budget proxy for max_visited** — bi-BFS caps `visited` via `max_visited = max(10, int(budget / 50))`. Uses L1 token-per-node coefficient (50) as the density ceiling, giving budget=500 → max_visited=10 and budget=5000 → max_visited=100. Balances completeness vs. bounded expansion (T-09.2-10).
- **estimate_exceeded preserves cardinality** — meta['cardinality_estimate'] is kept populated in the short-circuit response so agents can see WHY the query was rejected (counts + token estimate). Only text_body is empty.
