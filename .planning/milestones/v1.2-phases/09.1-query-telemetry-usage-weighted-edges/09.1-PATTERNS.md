**BFS/DFS return signature** (lines 251-280 — confirms edge tuple structure):
```python
def _bfs(G: nx.Graph, start_nodes: list[str], depth: int) -> tuple[set[str], list[tuple]]:
    visited: set[str] = set(start_nodes)
    frontier = set(start_nodes)
    edges_seen: list[tuple] = []
    for _ in range(depth):
        next_frontier: set[str] = set()
        for n in frontier:
            for neighbor in G.neighbors(n):
                if neighbor not in visited:
                    next_frontier.add(neighbor)
                    edges_seen.append((n, neighbor))
        visited.update(next_frontier)
        frontier = next_frontier
    return visited, edges_seen
```
Edge tuples are `(str, str)` — direction follows traversal order. Keys must be normalized: `f"{min(u,v)}:{max(u,v)}"` (graph is undirected `nx.Graph`, confirmed in `build.py` line 43).

**Weight formula** (no existing analog — new computation):
```python
import math

def _edge_weight(traversals: int) -> float:
    """Logarithmic weight formula: 1.0 + log(t), clamped to [1.0, 10.0]."""
    return max(1.0, min(10.0, 1.0 + math.log(max(1, traversals))))
```
Guard: `max(1, traversals)` prevents `math.log(0)`. Ceiling at 10.0 prevents runaway weights.

## Critical Implementation Notes

These are not patterns but constraints derived from reading source code that the planner must encode in task actions:

1. **Edge key normalization is mandatory** (`f"{min(u,v)}:{max(u,v)}"`) — `_bfs` and `_dfs` append `(n, neighbor)` tuples in traversal order; the graph is `nx.Graph` (undirected), so both `(A,B)` and `(B,A)` may appear. Without normalization, hot path counts halve and derived edge threshold never triggers. Confirmed: `build.py` line 43 uses `nx.Graph()`.

2. **`math.log(0)` guard** — counters must never reach zero when the weight formula runs. Apply `max(1, count)` inside `_edge_weight()`. During decay, entries that decay below 1 must be removed from the dict entirely rather than stored as 0 or 0.8.

3. **`usage_data` optional param placement** — append as the last parameter of `generate()`, after `suggested_questions`. This preserves positional compatibility with all existing call sites in `skill.md` (verified grep: called at lines 423, 484, 960).

4. **Derived edge duplication guard** — build the `existing` set from `_agent_edges` on every `_check_derived_edges` call, not from a cached structure. `_agent_edges` is loaded at startup and mutated in memory; re-reading from it is sufficient.

5. **`statistics.quantiles` small-data guard** — call `statistics.quantiles(data, n=10)` only when `len(data) >= 10`. Fall back to `max(data)` / `min(data)` for smaller lists to avoid `StatisticsError`.
