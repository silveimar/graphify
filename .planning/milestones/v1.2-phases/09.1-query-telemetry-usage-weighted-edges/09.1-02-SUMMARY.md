# Phase 09.1 Plan 02: Report Usage Patterns Summary

Percentile-based hot/cold edge classification in report.py with adaptive thresholds using statistics.quantiles for 10+ entries and max/min fallback for smaller datasets, rendering Usage Patterns section with Hot Paths and Cold Zones tables in GRAPH_REPORT.md.

## What Was Built

### Task 1: _compute_hot_cold helper (TDD)
- Added `import statistics` to report.py
- Implemented `_compute_hot_cold(G, counters, hot_percentile=0.90, cold_percentile=0.10)` returning dict with hot/cold edge lists, never_traversed count, and total_queries sum
- Uses `statistics.quantiles(all_counts, n=10)` for datasets with 10+ entries
- Falls back to max/min for smaller datasets to avoid StatisticsError
- 3 TDD tests: normal (12+ edges), empty counters, small data (<10 entries)

### Task 2: Usage Patterns section in generate()
- Extended `generate()` signature with `usage_data: dict | None = None` as last parameter
- Renders "## Usage Patterns" section with total traversals and never-traversed counts
- Hot Paths table (top 10%) and Cold Zones table (bottom 10%) with node labels
- All labels passed through `_sanitize_md()` for injection protection (T-09.1-05)
- Backward compatible: None or empty counters produce no section
- 4 tests: section presence, no-data backward compat, empty counters, label rendering
