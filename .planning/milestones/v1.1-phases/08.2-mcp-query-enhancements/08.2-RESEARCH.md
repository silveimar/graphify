## User Constraints (from CONTEXT.md)

### Locked Decisions

- **D-01:** `get_node` response appends three new lines to the existing text format: `Extracted At: {iso_timestamp}`, `Source Hash: {sha256}`, `Staleness: {FRESH|STALE|GHOST}`. No breaking change to return type — agents already parse the multi-line text.
- **D-02:** Staleness is computed live by calling `classify_staleness()` from `graphify.delta` on the node's data dict. This requires a `Path.exists()` + optional `stat()` call — negligible overhead for a single-node query.
- **D-03:** If a node has no provenance metadata (e.g., concept nodes, LLM-extracted nodes without source files), `get_node` shows `Extracted At: —`, `Source Hash: —`, `Staleness: FRESH` (matches `classify_staleness()` default behavior for missing provenance).
- **D-04:** New MCP tool `get_agent_edges` registered in `_handlers` dict alongside existing tools. Returns agent-edges from the in-memory `_agent_edges` list (loaded at startup from `agent-edges.json`).
- **D-05:** Filter parameters mirror `get_annotations` pattern (Phase 7 D-06): optional `peer_id` and `session_id` string filters. Returns all agent-edges when no filters provided.
- **D-06:** Return format is JSON array of edge records (same shape as stored in `agent-edges.json`). Consistent with `get_annotations` returning JSON.
- **D-07:** No time range filter for agent-edges — they are fewer and more permanent than annotations. Keep the tool simple. If time filtering is needed later, it can be added without breaking the interface.

## Summary

Phase 8.2 is a targeted two-feature extension to `serve.py`. There are no new modules, no new dependencies, and no architectural changes — only additive changes to a single file. Both features are powered by code that already exists in the codebase.

Feature 1: `get_node` provenance lines. The handler at `serve.py` §513-528 currently returns a 6-line text block. Three new lines are appended: `Extracted At`, `Source Hash`, and `Staleness`. The staleness value is computed by calling `classify_staleness(node_data)` from `graphify.delta` — a function that already exists (`delta.py` §68-89) and handles missing provenance by returning `FRESH`. The only change to `_tool_get_node()` is: import `classify_staleness`, read `extracted_at` and `source_hash` from the node data dict, call the classifier, and append three formatted lines.

Feature 2: `get_agent_edges` tool. The `_agent_edges` list is already loaded at server startup (`serve.py` §338) and updated by the `add_edge` tool. A new query tool follows the exact same structure as `_tool_get_annotations()` (`serve.py` §649-656): it reads optional `peer_id` and `session_id` arguments, filters the in-memory list, and returns `json.dumps(results)`. Tool registration follows the `get_annotations` inputSchema pattern. The `_handlers` dict at `serve.py` §658-671 gets one new entry.

**Primary recommendation:** Implement both features as direct extensions to `serve.py` using existing helper patterns. Total change is approximately 25-35 lines across one file plus one new filter helper function and corresponding tests.

## Architecture Patterns

### Existing Pattern: Tool Handler Function

Every MCP tool in `serve.py` is a plain sync function (no `async`) stored in the `_handlers` dict. Handlers return a `str` — either plain text or `json.dumps(...)`. [VERIFIED: codebase read of `serve.py` §498-671]

```python
# Source: serve.py §649-656 — existing get_annotations pattern
def _tool_get_annotations(arguments: dict) -> str:
    peer_id = arguments.get("peer_id") or None
    session_id = arguments.get("session_id") or None
    time_from = arguments.get("time_from") or None
    time_to = arguments.get("time_to") or None
    results = _filter_annotations(_annotations, peer_id, session_id, time_from, time_to)
    return json.dumps(results)
```

`get_agent_edges` MUST follow this exact pattern.

### Existing Pattern: Filter Helper Function

`_filter_annotations()` (`serve.py` §172-189) is a module-level helper with signature:

```python
def _filter_annotations(
    annotations: list[dict],
    peer_id: str | None,
    session_id: str | None,
    time_from: str | None,
```

A parallel `_filter_agent_edges()` helper must be added at module level for testability (same reason `_filter_annotations` is at module level, not nested inside the tool — `tests/test_serve.py` imports helpers directly). [VERIFIED: codebase read of `test_serve.py` §7-25]

### Existing Pattern: Import of `classify_staleness`

`delta.py` uses a lazy import of `cache.file_hash` inside `classify_staleness()` itself (`delta.py` §92: `from .cache import file_hash`). This means callers need only `from graphify.delta import classify_staleness` at the top of `serve.py`. [VERIFIED: codebase read of `delta.py` §68-98]

The import question in Claude's Discretion resolves clearly: add `from graphify.delta import classify_staleness` to the top-level imports of `serve.py` alongside the existing `from graphify.security import sanitize_label`. Top-level is simpler and consistent.

### Recommended: Add `node_id` filter to `get_agent_edges`

The edge record schema (`_make_edge_record`) stores `source` and `target` fields. [VERIFIED: codebase read of `serve.py` §105-116]. A `node_id` filter — matching edges where `source == node_id OR target == node_id` — is genuinely useful (agents query "all edges I added for node X") and adds ~2 lines to the filter helper. Recommend including it.

## Common Pitfalls

### Pitfall 1: Nesting helper functions (kills testability)
**What goes wrong:** `_filter_agent_edges` defined inside `serve()` closure cannot be imported by `test_serve.py`.
**Why it happens:** Following the `_tool_*` handler pattern where those functions ARE closure-nested.
**How to avoid:** Keep filter helpers (`_filter_annotations`, `_filter_agent_edges`) at module level. Only the `_tool_*` handlers live inside the `serve()` closure because they need closure-scoped `_agent_edges`, `_annotations`, etc.
**Warning signs:** If `_filter_agent_edges` is defined after `def _reload_if_stale():`, it's in the closure.

### Pitfall 2: Forgetting to export new helper from test imports
**What goes wrong:** `tests/test_serve.py` imports a fixed list of helpers. New functions (`_filter_agent_edges`) must be added to the import block.
**Why it happens:** Python imports require explicit names.
**How to avoid:** Add `_filter_agent_edges` to the import list in `test_serve.py` alongside `_filter_annotations`.
**Warning signs:** `ImportError` on the test import line.

### Pitfall 3: Breaking get_node return format
**What goes wrong:** Returning a dict or restructuring the text format breaks agents already parsing the 6-line output.
**Why it happens:** Attempting to return richer structured data.
**How to avoid:** D-01 is locked: append three lines to the existing `"\n".join([...])` list. Do not change the format of existing lines.
**Warning signs:** Any change to lines 1-6 of the get_node output.

### Pitfall 4: `classify_staleness` receives wrong argument type
**What goes wrong:** Passing `nid` (the node ID string) instead of `d` (the node data dict).
**Why it happens:** Both are in scope from the `_tool_get_node` unpacking: `nid, d = matches[0]`.
**How to avoid:** Call `classify_staleness(d)`, not `classify_staleness(nid)`.
**Warning signs:** `classify_staleness` returning `"FRESH"` for all nodes because `d.get("source_file", "")` returns empty string.

### Pitfall 5: `_agent_edges` list not accessible in closure-nested tool handler
**What goes wrong:** `_tool_get_agent_edges` references `_agent_edges` but it's a local variable inside `serve()`.
**Why it happens:** Python closures capture by reference — this actually works correctly for mutable list objects.
**How to avoid:** Follow the same pattern as `_tool_get_annotations` which captures `_annotations`. `_agent_edges` is already captured correctly by other handlers (`_tool_add_edge` uses it at §640).
**Warning signs:** `NameError: name '_agent_edges' is not defined` — would only occur if the function were NOT nested inside `serve()`.

## Code Examples

### Extending `_tool_get_node` (verified pattern)

```python
# Source: serve.py §513-528 — current + new lines
from graphify.delta import classify_staleness  # add to module-level imports

def _tool_get_node(arguments: dict) -> str:
    _reload_if_stale()
    label = arguments["label"].lower()
    matches = [(nid, d) for nid, d in G.nodes(data=True)
               if label in d.get("label", "").lower() or label == nid.lower()]
    if not matches:
        return f"No node matching '{label}' found."
    nid, d = matches[0]
    staleness = classify_staleness(d)
    extracted_at = d.get("extracted_at", "—")
    source_hash = d.get("source_hash", "—")
    return "\n".join([
        f"Node: {d.get('label', nid)}",
        f"  ID: {nid}",
        f"  Source: {d.get('source_file', '')} {d.get('source_location', '')}",
        f"  Type: {d.get('file_type', '')}",
        f"  Community: {d.get('community', '')}",
        f"  Degree: {G.degree(nid)}",
        f"  Extracted At: {extracted_at}",
        f"  Source Hash: {source_hash}",
        f"  Staleness: {staleness}",
    ])
```

### Module-level filter helper (verified pattern)

```python
# Pattern mirrors _filter_annotations at serve.py §172-189
def _filter_agent_edges(
    edges: list[dict],
    peer_id: str | None,
    session_id: str | None,
    node_id: str | None,
) -> list[dict]:
    """Filter agent-edge list by optional peer_id, session_id, or node_id (source or target)."""
    result = list(edges)
    if peer_id is not None:
        result = [e for e in result if e.get("peer_id") == peer_id]
    if session_id is not None:
        result = [e for e in result if e.get("session_id") == session_id]
    if node_id is not None:
        result = [e for e in result if e.get("source") == node_id or e.get("target") == node_id]
    return result
```

### Known Threat Patterns

| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| Path traversal via `node_id` filter value | Tampering | `node_id` is only used for equality comparison, never as a path — no risk |

| `peer_id`/`session_id` filter injection | Tampering | String equality comparison only — no SQL, no eval, no path. No sanitization needed for filter params. |

## Project Constraints (from CLAUDE.md)

- Python 3.10+ — no walrus operator without `from __future__ import annotations` guard (already present in all modules)
- No new required dependencies — `classify_staleness` import is intra-package (`graphify.delta`), not a new PyPI dependency
- Pure unit tests — no network calls, no filesystem side effects outside `tmp_path`
- Test file naming: `tests/test_serve.py` (already exists, extend it)
- Module docstring and `from __future__ import annotations` at top of every module
- Type hints on all functions: `str | None` (not `Optional[str]`), `list[dict]` (not `List[dict]`)
- Leading underscore for private/internal module-level helpers: `_filter_agent_edges`
- Warning/error messages to stderr with `[graphify]` prefix if needed
- All external string inputs through `sanitize_label()` — filter param values are not stored, only compared, so sanitization is not required there (consistent with `_filter_annotations` which does not sanitize filter params)
