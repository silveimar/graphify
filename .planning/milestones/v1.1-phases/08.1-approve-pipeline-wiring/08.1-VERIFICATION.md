### Human Verification Required

#### 1. User-Modified Detection in Live Vault (Approve Path)

**Test:** Set up a vault with at least one note written by `graphify --obsidian`. Manually edit that note to add user content. Run `graphify approve <id> --vault <vault_path>` where the proposal targets that modified note.
**Expected:** stderr prints `[graphify] Note 'NoteTitle.md' was user-modified, skipping. Use --force to override.` The proposal file remains in `graphify-out/proposals/`. Running with `--force` writes the note and deletes the proposal file.
**Why human:** Requires a live vault with a populated `vault-manifest.json` from a prior `--obsidian` run. The hash comparison reads actual on-disk content; cannot be exercised by pytest without a full vault fixture.

#### 2. Auto Snapshot + Delta on Subsequent `/graphify` Run

**Test:** Run `/graphify` on a corpus that already has a snapshot in `graphify-out/snapshots/`. Observe output and output directory.
**Expected:** A new snapshot JSON appears in `graphify-out/snapshots/`, `GRAPH_DELTA.md` is updated, and skill output includes `Snapshot saved (N nodes). Delta: see GRAPH_DELTA.md`.
**Why human:** `skill.md` Python blocks execute inside a Claude agent session; they are not importable or callable by pytest.

#### 3. Auto Snapshot on First `/graphify` Run

**Test:** Delete `graphify-out/snapshots/` (or use a fresh project). Run `/graphify`.
**Expected:** `graphify-out/snapshots/` is created with one snapshot file, `GRAPH_DELTA.md` is written with first-run content, and output includes `Snapshot saved (N nodes). No prior snapshot for delta comparison.`
**Why human:** Same as above — skill.md execution requires live agent invocation.
