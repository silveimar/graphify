# Phase 08.1 Plan 01: Manifest-Aware Approve Pipeline Summary

Thread vault-manifest.json through the `graphify approve` path so approved proposals respect Phase 8's user-modified detection, add `--force` bypass, emit SKIP_PRESERVE warning, and delete proposal files on success.

## What Was Built

The `graphify approve` subcommand previously bypassed Phase 8's manifest-based user-modified detection entirely (INT-01 gap). After this plan, the approve flow has identical protection to `graphify --obsidian` re-runs.

### Changes to `graphify/__main__.py`

**New indirection helper:**
- `_load_manifest_for_approve(manifest_path)` — wraps `merge._load_manifest`; allows tests to monkeypatch manifest loading independently

**Updated `_apply_merge_plan_for_approve` signature:**
- Now accepts `*, manifest_path=None, old_manifest=None` keyword args and forwards them to `apply_merge_plan`

**Updated `_approve_and_write_proposal`:**
- Added `force: bool = False` keyword parameter
- Loads `graphify-out/vault-manifest.json` before merge calls via `_load_manifest_for_approve`
- Passes `manifest=old_manifest, force=force` to `_compute_merge_plan_for_approve`
- Passes `manifest_path=manifest_path, old_manifest=old_manifest` to `_apply_merge_plan_for_approve`
- Checks result for `SKIP_PRESERVE` actions with `user_modified=True`; emits stderr warning and returns early without deleting proposal (D-04)
- Deletes proposal file via `path.unlink()` on success (D-05) — replaces the old status-rewrite pattern

**Arg parser:**
- Added `force = False` initializer
- Added `elif args[i] == "--force":` branch
- Threaded `force=force` through both call sites (batch `--all` and single `<id>`)

### Changes to `tests/test_approve.py`

Updated `test_approve_and_write_proposal_calls_merge`:
- `fake_apply_merge_plan` now accepts `*, manifest_path=None, old_manifest=None`
- Assertion changed from checking `on_disk["status"] == "approved"` to verifying proposal file deleted

Added 5 new tests:
- `test_approve_deletes_proposal` — proposal JSON removed from disk after successful approve
- `test_approve_writes_manifest` — `manifest_path` and `old_manifest` kwargs threaded to apply call
- `test_approve_skips_user_modified` — proposal stays on disk when SKIP_PRESERVE+user_modified
- `test_approve_user_modified_warning` — stderr contains "was user-modified, skipping" and "--force"
- `test_approve_force_flag` — `force=True` forwarded to compute_merge_plan kwargs
