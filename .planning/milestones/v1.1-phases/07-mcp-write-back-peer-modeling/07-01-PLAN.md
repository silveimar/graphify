<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client -> serve.py tools | Agent-supplied strings (node_id, text, relation, peer_id) are untrusted input |
| serve.py -> graphify-out/ | Write operations to sidecar files must not escape output directory |
| serve.py -> G (graph object) | Mutation tools must never modify the in-memory graph |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | _tool_annotate_node | mitigate | All string inputs sanitized via `sanitize_label()` before persistence — strips control chars, caps length |
| T-07-02 | Information Disclosure | peer_id default | mitigate | Default to string literal `"anonymous"`; never read `os.environ["USER"]` or `socket.gethostname()` |
| T-07-03 | Tampering | _tool_add_edge | mitigate | Edge written ONLY to `agent-edges.json` sidecar; `G.add_edge()` never called in any tool handler |
| T-07-04 | Denial of Service | annotations.jsonl | mitigate | JSONL compaction at server startup deduplicates records; single-server assumption (D-16) prevents unbounded growth from concurrent writers |
| T-07-05 | Information Disclosure | session_id | mitigate | Use `uuid.uuid4()` exclusively — never `uuid.uuid1()` which embeds MAC address |
| T-07-06 | Tampering | annotations.jsonl | accept | Corrupt lines from crash mid-write are skipped during compaction; data loss limited to at most one record |
</threat_model>
