2. Add `_parse_user_sentinel_blocks(body: str) -> list[tuple[int, int, str]]` function.
   Returns list of (start_line_idx, end_line_idx, content_between_sentinels) tuples.
   Per D-03 (Claude's Discretion): support multiple USER_START/END pairs — simpler for users and equally simple to implement.
   Algorithm:
   - Split body into lines
   - Scan lines for START markers, then find matching END marker
   - If START found without matching END after it: print `"[graphify] malformed user sentinel: START without matching END — ignoring user blocks"` to stderr, return []
   - If nested START (another START before finding END): print `"[graphify] malformed user sentinel: nested START markers — ignoring user blocks"` to stderr, return []
   - Collect (start_line_idx, end_line_idx, content) for each valid pair where content is the lines BETWEEN start and end (exclusive of sentinel lines)
   - Return the list (empty list = no user blocks)

3. Add `_has_user_sentinel_blocks(body: str) -> bool` convenience helper:
   ```python
   def _has_user_sentinel_blocks(body: str) -> bool:
       return bool(_USER_SENTINEL_START_RE.search(body) and _USER_SENTINEL_END_RE.search(body))
   ```
   Used by _build_manifest_from_result to set has_user_blocks without full parsing.

4. Add `_extract_user_blocks(text: str) -> list[str]` function:
   Extract the full sentinel block strings (including markers) from the file text.
   Returns list of strings like `"<!-- GRAPHIFY_USER_START -->\nuser content\n<!-- GRAPHIFY_USER_END -->"`.
   Used before rewriting to capture user content.

5. Add `_restore_user_blocks(new_text: str, user_blocks: list[str]) -> str` function:
   If new_text contains user sentinel markers (empty pairs from template), replace them with the captured user blocks in order. If new_text has no sentinel markers but user_blocks is non-empty, append user blocks at the end of the body (before any trailing newline). This ensures user content survives even if the template structure changes.

   - For UPDATE action: same pattern — extract user blocks from existing_text before merge, restore after merge. The _merge_body_blocks handles graphify-managed sentinels; user sentinels are a separate layer.
   - For CREATE action: no change needed (no existing file to extract from).
   ```python
   # In the CREATE/REPLACE branch:
   if action.action in ("CREATE", "REPLACE"):
       fm_text = _dump_frontmatter(rendered_note["frontmatter_fields"])

       # D-08: preserve user sentinel blocks from existing file during REPLACE
       if action.action == "REPLACE" and existing_text is not None:
           user_blocks = _extract_user_blocks(existing_text)

               return _restore_user_blocks(result_text, user_blocks)
       if body.startswith("\n"):
           return fm_text + body
       return fm_text + "\n" + body
   ```
