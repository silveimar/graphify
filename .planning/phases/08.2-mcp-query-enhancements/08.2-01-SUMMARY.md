## What Was Built

### Task 1: get_node provenance lines (D-01, D-02, D-03)

`_tool_get_node()` now returns a 9-line text block instead of 6 lines. The three new lines are appended after `Degree`:

```
  Extracted At: 2025-01-01T00:00:00+00:00
  Source Hash: abc123
  Staleness: FRESH
```

- `classify_staleness(d)` is called live on the node data dict (not the node ID)
- Missing provenance fields (`extracted_at`, `source_hash`) default to em-dash (`—`)
- `classify_staleness` returns `FRESH` for nodes without provenance — so missing = FRESH (D-03)
- Import added at module level: `from graphify.delta import classify_staleness`

### Task 2: get_agent_edges tool (D-04, D-05, D-06, D-07)

New module-level helper `_filter_agent_edges(edges, peer_id, session_id, node_id)`:
- Filters by `peer_id` (exact match)
- Filters by `session_id` (exact match)
- Filters by `node_id` (matches source OR target — bidirectional)
- All filters are optional (pass `None` to skip)
- Filters are AND-combined (intersection)

New MCP tool `get_agent_edges` registered in `list_tools` with:
- `inputSchema.properties`: `peer_id`, `session_id`, `node_id` (all optional, no `required` key)
- No time range filter (D-07)

New handler `_tool_get_agent_edges` added to closure and `_handlers` dict.
