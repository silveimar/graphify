<objective>
Fix the auto-adopt argparse defect: `--vault required=True` on the `update-vault` and `vault-promote` subparsers fires before the post-parse auto-adopt fallback can inject `Path.cwd()`, so running these commands from a vault CWD without `--vault` exits 2 with `argparse: required: --vault` instead of auto-adopting.

Purpose: Restore the intended VCWD-02 auto-adopt UX for the two commands that actually route to a vault. Locked design (approach b) is documented in `.planning/debug/vault-cwd-gate-argparse-required.md`.
Output: Two argparse declarations flipped to `required=False, default=None`; existing post-parse guard tightened with a friendly-error else branch; two RED tests unskipped (now GREEN); one new test added for the friendly-error branch.
</objective>
