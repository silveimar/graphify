From graphify/merge.py (the exact patterns to lift):
```python
def _content_hash(path: Path) -> str:
    return hashlib.sha256(Path(path).read_bytes()).hexdigest()

def _write_atomic(target: Path, content: str) -> None:
    tmp = target.with_suffix(target.suffix + ".tmp")
    target.parent.mkdir(parents=True, exist_ok=True)
    try:
        with open(tmp, "w", encoding="utf-8") as fh:
            fh.write(content); fh.flush(); os.fsync(fh.fileno())
        os.replace(tmp, target)
    except OSError:
        if tmp.exists():
            try: tmp.unlink()
            except OSError: pass
        raise
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argv → filesystem | `--vault PATH` is untrusted user input; must be confined |
| graph.json `source_file` → filename | Used only to build relative paths; routed through safe_filename |
| vault → .graphify/profile.yaml write | Write-back mutates vault-owned file — must re-validate on every write |
| graphify-out/ → vault-manifest.json | Sidecar write; confined via security.validate_graph_path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-03-01 | Tampering | Path traversal via `--vault ../../etc` | mitigate | `profile.validate_vault_path(vault_path, vault_path)` at entry resolves symlinks and confines; every subsequent write re-validates `vault_dir / rel_path` against `vault_dir` |
| T-19-03-02 | Tampering | Symlink escape from a subfolder (e.g. `Atlas/Dots/Things` symlinked to `/tmp`) | mitigate | `validate_vault_path` uses `.resolve()` on both sides per profile.py implementation; symlink escapes rejected |
| T-19-03-03 | Tampering | Overwriting a user-edited note | mitigate | Hash-match-or-skip decision table (D-13) — if disk bytes don't match last-written hash, file is considered user-modified and skipped |
| T-19-03-04 | Tampering | Foreign file overwrite (file exists but is not in manifest) | mitigate | `write_note` returns `"skipped_foreign"` when path not in manifest and file present on disk; never writes |
| T-19-03-05 | Tampering | YAML injection in profile.yaml write-back via node label masquerading as tag | mitigate | Layer 3 values flow from closed `_TECH_EXT_MAP` (Plan 02); `yaml.dump` never takes string-concat input; pre-existing profile values pass through `yaml.safe_load` then `yaml.dump` (no re-encoding bypass) |
| T-19-03-06 | Tampering | Manifest file tampering to force overwrite of foreign file | accept | Manifest is under graphify-out/ (local-only, same trust domain as the user's own toolchain); if the user's local machine is compromised, the threat model doesn't extend further |
| T-19-03-07 | Information Disclosure | `import-log.md` leaking absolute vault paths | accept | Log is for the operator's own debugging; absolute paths are expected and not sensitive in a local CLI tool |
| T-19-03-08 | Denial of Service | Huge graph with 100k+ nodes during promote() | accept | Linear in node count; same complexity as existing `--obsidian` handler |
| T-19-03-09 | Elevation of Privilege | Running vault-promote with elevated perms writes into system dirs | accept | Out of scope — user runs CLI with their own permissions |
</threat_model>
