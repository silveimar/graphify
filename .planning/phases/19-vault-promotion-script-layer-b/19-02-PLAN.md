<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| graph.json → in-memory G | Graph content is user-generated (LLM-extracted labels); labels may contain control chars, quotes, YAML-breaking characters |
| user profile.yaml → taxonomy merge | User-controlled Layer 2 input merges into runtime profile; must be validated upstream (Plan 01) |
| node.source_file → tech-tag detection | Extension derived from arbitrary path strings; never written to disk in this plan |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-02-01 | Tampering | YAML injection via node label in frontmatter | mitigate | All frontmatter string values flow through `profile.safe_frontmatter_value` (already hardened); all tag values flow through `profile.safe_tag` |
| T-19-02-02 | Tampering | Node label used as wikilink target (`[[...]]`) could embed `]]\nuser_payload\n[[` | mitigate | Wikilink stem produced by `safe_filename` (NFC normalize + strip OS-illegal chars including `]`); display text kept verbatim inside `safe_frontmatter_value` quoting |
| T-19-02-03 | Tampering | `_detect_tech_tags` consuming attacker-crafted source_file paths to inject unexpected tags | mitigate | Extension lookup is a closed map (`_TECH_EXT_MAP`); unknown extensions map to nothing; output passed through `safe_tag` |
| T-19-02-04 | Information Disclosure | Rendering an INFERRED/AMBIGUOUS edge into `related:` would leak low-confidence links as authoritative | mitigate | Hard filter `d.get("confidence") == "EXTRACTED"` in `_extracted_neighbors`; unit-tested explicitly |
| T-19-02-05 | Tampering | Template placeholder injection via `${...}` tokens embedded in labels | mitigate | `string.Template.safe_substitute` only resolves placeholders from the substitution dict — labels passed as values are not re-evaluated |
| T-19-02-06 | Denial of Service | Huge graph causing render loop OOM | accept | Linear in nodes; render happens one note at a time (streaming caller in Plan 03); no quadratic algorithm |
</threat_model>
