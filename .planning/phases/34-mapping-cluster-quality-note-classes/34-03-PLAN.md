<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo identity -> CODE filename | CLI/profile/git/directory repo identity affects output paths. |
| node labels/source files -> CODE filename | Extracted labels and source paths can contain path separators, control characters, or collision-prone names. |
| export render dispatch -> vault merge plan | Rendered filenames become merge-plan paths and later vault writes. |
| legacy community requests -> MOC output | Legacy `community` requests must be coerced without writing `_COMMUNITY_*`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-03-01 | Tampering | `graphify.naming.build_code_filename_stems` | mitigate | Normalize repo identity with `normalize_repo_identity()` and node labels with existing filename normalization; never concatenate raw labels into paths. |
| T-34-03-02 | Spoofing / Collision | CODE filename groups | mitigate | Group by base stem and suffix every colliding member with `sha256(node_id + "\0" + source_file)[:8]` so output is order-independent. |
| T-34-03-03 | Tampering | `graphify/export.py` merge-plan paths | mitigate | Inject only filename stems into contexts; keep path joining in existing render/merge path confinement helpers. |
| T-34-03-04 | Tampering | CODE filename render support | mitigate | Consume only the sanitized `filename_stem` generated by `graphify.naming`; keep frontmatter/wiki metadata rendering in Plan 04 through existing helpers. |
| T-34-03-05 | Tampering / Data loss | legacy `_COMMUNITY_*` handling | mitigate | Coerce `note_type: community` to `render_moc()` with warning and do not scan/delete existing vault legacy files. |
</threat_model>
