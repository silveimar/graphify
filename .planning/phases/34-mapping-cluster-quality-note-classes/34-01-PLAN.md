## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| profile.yaml -> profile validation | User-authored note types, dataview keys, taxonomy folders, and mapping settings enter trusted export configuration. |
| node labels -> templates/frontmatter/wikilinks | Graph labels and source metadata can contain unsafe Markdown/YAML/wiki characters. |
| legacy community inputs -> renderer | Old `note_type: community` requests must not revive `_COMMUNITY_*` overview writes. |
| legacy vault files -> Phase 34 execution | Existing `_COMMUNITY_*` files in user vaults are user data and must not be scanned, edited, or deleted in this phase. |

| T-34-01-03 | Tampering / Data loss | legacy `_COMMUNITY_*` vault notes | mitigate | Do not add any filesystem scan or deletion for legacy files per D-04; Phase 35 owns migration candidates and orphans. |
| T-34-01-04 | Spoofing / Collision | future CODE filenames | transfer | This plan only establishes the `code` type; Plan 03 must implement deterministic collision suffixes before export writes CODE paths. |
