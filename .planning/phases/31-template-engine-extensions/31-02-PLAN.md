<objective>
Add per-note-type Dataview query strings (TMPL-03) declared via a new top-level `dataview_queries: {note_type: query_string}` profile key. `_build_dataview_block` consults the per-note-type override first and falls back to today's `obsidian.dataview.moc_query` when no entry is declared. `validate_profile` restricts keys to `_KNOWN_NOTE_TYPES = {moc, community, thing, statement, person, source}`. The new key composes cleanly with Phase 30's `extends:`/`includes:` chain via `_deep_merge`, and per-key provenance surfaces in `--validate-profile` output.

Purpose: Ship the third v1.7 template-engine extension while preserving backward compatibility (profiles without `dataview_queries` render exactly as today) and the Phase 30 graph-blind preflight discipline (unknown keys fail at load time, never mid-render).

Output: Extended `graphify/profile.py` (whitelist + validator + provenance) and `graphify/templates.py::_build_dataview_block` (per-note-type lookup), plus comprehensive tests in `tests/test_profile.py` and `tests/test_templates.py`.
</objective>
