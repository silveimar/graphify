<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-corpus → extract → build | Untrusted source code / docs become edge attrs; relation/source_file strings flow into temporal-stamped edges |
| filesystem → load_decay_config | Package-shipped YAML; user could replace if they edit site-packages |
| env-var → run_now_iso | GRAPHIFY_RUN_TS pins clock; non-ISO values could propagate downstream |
| filesystem → stamp_supersessions prior graph.json | Untrusted file content; malformed JSON or unexpected fields possible |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-71-01 | Tampering | temporal_config.yaml parse | mitigate | Use `yaml.safe_load` (never `yaml.load`); guard ImportError, FileNotFoundError, YAMLError → fall back to in-code defaults. Documented in CLAUDE.md as package-shipped not user-config. |
| T-71-02 | Information Disclosure / Input Validation | malformed valid_from in legacy edge | mitigate | `compute_decay_weight` only called when stamping at build time on edges we control; if downstream code reads malformed valid_from, `datetime.fromisoformat` raises ValueError — read-mode validator tolerates absence but new code paths must catch ValueError and treat as legacy (decay_weight=1.0). |
| T-71-03 | Spoofing | GRAPHIFY_RUN_TS env injection | accept | Documented as test-only override; non-ISO value will fail downstream `datetime.fromisoformat` calls — fail loud. Production runs do not set this var. |
| T-71-04 | Denial of Service | very large temporal_config.yaml | accept | File is package-shipped; size bounded by repo. yaml.safe_load has no recursion-bomb path with safe loader on flat config. |
| T-71-05 | Tampering | adding valid_from to REQUIRED_EDGE_FIELDS would break legacy compat | mitigate | Explicitly forbidden in task action; write-mode validator handles strict check separately. Legacy-fixture test gates this. |
| T-71-13 | Tampering | malformed prior graph.json passed to stamp_supersessions | mitigate | Catch JSONDecodeError / KeyError / nx errors and return new_edges unchanged (treat as "no prior"). Tested via test_supersession_malformed_prior. |
</threat_model>
