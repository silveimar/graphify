# Phase 60.1 Plan 03: Note-Type Precedence + Render-Override Ladder Gap-Closure Summary

Closed **APPLY-DET-01** verification gaps #1 and #2 by fixing two latent bugs in the
`update-vault --apply` pipeline that the Wave 1 determinism fix (Plan 60.1-01)
exposed end-to-end via the locked Phase 60 RED test
`tests/test_e2e_integration.py::test_e2e_compose_override_ladder` (commit `333d2da`):

1. **Classification precedence bug** in `graphify/mapping.py` topology fallback —
   code-backed god nodes were always classified as `note_type="code"`, ignoring
   the active vault profile's `note_type_templates` override surface.

2. **Render-time override-loader bugs** in `graphify/templates.py` and the
   `update-vault` plumbing — non-MOC override templates (e.g. `thing` overrides
   declared via `note_type_templates`) were silently rejected because the loader
   validated every override against MOC's required-slot set, and even when a
   template would have been accepted the vault root was never threaded through
   to `render_note`/`render_moc` so `<vault>/.graphify/templates/` lookups
   resolved against `notes_dir`, not the actual vault.

## Patches

### 1. `graphify/mapping.py` — topology fallback honors `note_type_templates`

**Before** (lines 414-422):
```python
if node_id in god_ids:
    if _is_code_note_candidate(G, node_id, god_ids):
        note_type = "code"
    else:
        note_type = "thing"
else:
    note_type = "statement"
folder = _resolve_folder(note_type, None, folder_mapping)
```

**After** (additions in `classify()` setup + topology branch):
```python
# Phase 60.1-03 (APPLY-DET-01): collect the set of note_type slugs the
# active profile declares via `note_type_templates[].pattern`. ...
override_note_types: set[str] = set()
for _tpl in profile.get("note_type_templates") or []:
    if isinstance(_tpl, dict):
        _pat = _tpl.get("pattern")
        if isinstance(_pat, str):
            override_note_types.add(_pat)
...
if node_id in god_ids:
    if _is_code_note_candidate(G, node_id, god_ids):
        if "thing" in override_note_types:
            note_type = "thing"
        else:
            note_type = "code"
    else:
        note_type = "thing"
else:
    note_type = "statement"
```

### 2. `graphify/templates.py` — override loader validates against target note_type

**Before** (line 1585):
```python
errors = validate_template(text, _REQUIRED_PER_TYPE["moc"])
```

**After** (parameterised via new `required_note_type` kwarg, threaded from Tier 3
`note_type_templates` and Tier 1 `mapping_rule_templates` resolvers):
```python
def _load_override_template(
    rel_path, vault_dir, default_template, *,
    list_name="community_templates",
    required_note_type="moc",
):
    ...
    _required_key = required_note_type if required_note_type in _REQUIRED_PER_TYPE else "moc"
    errors = validate_template(text, _REQUIRED_PER_TYPE[_required_key])
```

Tier 3 (`note_type_templates`) passes `required_note_type=note_type`; Tier 1
(`mapping_rule_templates`) passes the resolved `note_type` of the matched
mapping rule. Tier 2 (`community_templates`) keeps the legacy `"moc"` default
since community templates are MOC-shaped.

### 3. `graphify/export.py` — vault root threaded into renderers

**Before**:
```python
filename, rendered_text = render_note(
    node_id, G, profile, note_type, ctx, vault_dir=out,
)
```

`out` is `notes_dir` (e.g. `<vault>/Atlas/Sources/Graphify`), not the vault
root, so the override-template loader looked at
`<notes_dir>/.graphify/templates/...` and silently fell back to the builtin.

**After**: `to_obsidian` accepts an optional `vault_root: Path | None`; when
provided, `_override_root = Path(vault_root).resolve()` is threaded into
`render_note` and `render_moc` calls. Legacy callers (no `vault_root`) keep
the previous behavior — `_override_root` falls back to `out`.

### 4. `graphify/migration.py` — pass vault root through `update-vault`

```python
render_context = to_obsidian(
    G, communities, str(resolved.notes_dir),
    profile=profile, ...,
    vault_root=vault,   # <-- the actual vault path validated upstream
)
```

### 5. `graphify/vault_promote.py` — render-override-aware path (defensive)

`vault_promote.render_note` is on a sibling code path (the legacy 7-bucket
promotion pipeline). Threaded `vault_dir` through it for parity, even though
the `update-vault` flow uses `templates.render_note` via `export.to_obsidian`
and is what the locked E2E exercises. No behavior change for existing callers
that omit `vault_dir`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Render-time override-template loader rejected non-MOC overrides**
- **Found during:** Task 3 (locked E2E run after Task 2 GREEN)
- **Issue:** `_load_override_template` in `graphify/templates.py` validated every
  override against `_REQUIRED_PER_TYPE["moc"]` (`{frontmatter, label, members_section, dataview_block}`),
  which silently rejected the test's `templates/ntt-thing.md` (which only declares
  `frontmatter` + `label`, the correct contract for a `thing` template per
  `_REQUIRED_PER_TYPE["thing"] = {"frontmatter", "label"}`). The loader fell
  through to the base builtin without surfacing a stderr warn (subprocess
  swallowed it during the E2E test).
- **Fix:** Added `required_note_type` kwarg to `_load_override_template`; Tier 3
  (`note_type_templates`) passes the target `note_type`, Tier 1
  (`mapping_rule_templates`) passes the resolved `note_type`, Tier 2 keeps
  legacy `"moc"` default for community templates.
- **Files modified:** `graphify/templates.py`
- **Commit:** (squashed into the Plan 03 GREEN commit chain)

**2. [Rule 3 - Blocking] Vault root not threaded into `render_note`/`render_moc`**
- **Found during:** Task 3 (locked E2E still failing after fix #1)
- **Issue:** `export.to_obsidian` passes `vault_dir=out` (which is `notes_dir`,
  e.g. `<vault>/Atlas/Sources/Graphify`) into `render_note`. The override loader
  resolves overrides against `<vault_dir>/.graphify/...`, so the lookup happened
  under `<notes_dir>/.graphify/`, which doesn't exist. Override always fell
  through to default.
- **Fix:** Added optional `vault_root: Path | None` kwarg to `to_obsidian`;
  `migration.run_update_vault` threads the actual vault path through. When
  provided, the override-template lookup root is the vault root; when omitted
  (legacy callers), behavior is unchanged.
- **Files modified:** `graphify/export.py`, `graphify/migration.py`,
  `graphify/vault_promote.py`
- **Commit:** (squashed into the Plan 03 GREEN commit chain)

Both deviations were essential to flip the locked Phase 60 E2E test green;
without them, fix #1 (mapping.py) would have correctly emitted `type: thing`
in frontmatter (proven by the RED→GREEN unit test) but the rendered note bodies
would still come from the base builtin template, missing the
`OVERRIDE_NTT_THING_MARKER` sentinel that the locked test asserts on.
