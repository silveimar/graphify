## Domain framing

Phase 60.1 is a surgical determinism fix: `graphify update-vault --apply --plan-id <id>` currently fails on first run because preview and apply produce different `plan_id`s on identical input. Memory 4739 traced this to `graspologic.leiden()` running without `random_seed` in `graphify/cluster.py:_partition`. The Louvain fallback at `cluster.py:48` already uses `seed=42`, so the Leiden path is the outlier.

Phase 60's RED test `tests/test_e2e_integration.py::test_e2e_compose_override_ladder` (commit `333d2da`) is locked as the regression gate — it must turn green without modification.
