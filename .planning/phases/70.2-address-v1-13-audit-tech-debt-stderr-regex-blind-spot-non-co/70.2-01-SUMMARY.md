# Phase 70.2 Plan 01: Stderr Contract Regex Blind Spot Summary

Tightened the stderr-contract regex so f-strings with single-line nested parens (e.g. `({len(real_errors)} issues)`) can no longer slip past `test_no_outlier_stderr_prefixes_in_source`, then rewrote every newly-surfaced non-compliant `print(..., file=sys.stderr)` call to the `[graphify] error:/info:/hint:` whitelist defined by D-04 / D-10.

## What Changed

- **tests/test_stderr_contract.py** — `test_no_outlier_stderr_prefixes_in_source` now greps `print\(.*file=sys\.stderr` (greedy `.*` instead of `[^)]*`). Per-line scope is preserved (grep matches per-line), so multi-line print calls remain out of scope of this test exactly as before.
- **graphify/build.py:251** — extraction-warning emission now starts with `[graphify] hint: ` (D-10).
- **graphify/naming.py** — four `repo identity` emissions (cli-flag, profile, fallback-git-remote, fallback-directory) now start with `[graphify] info: `.
- **tests/test_naming.py, tests/test_main_flags.py, tests/test_main_cli.py** — assertions updated to match the new compliant text.
