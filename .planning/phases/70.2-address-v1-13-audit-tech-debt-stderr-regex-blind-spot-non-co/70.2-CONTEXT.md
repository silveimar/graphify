# Phase 70.2: Address v1.13 Audit Tech Debt - Context

**Gathered:** 2026-05-06
**Status:** Ready for planning

<domain>
## Phase Boundary

Close three specific tech-debt items surfaced by the v1.13 milestone audit (`v1.13-MILESTONE-AUDIT.md`):

1. **AUDIT-02 follow-up** — `build.py:251` emits a non-D-04-compliant stderr line (`[graphify] Extraction warning (...)`) that is invisible to `tests/test_stderr_contract.py::test_no_outlier_stderr_prefixes_in_source` because the grep regex `[^)]*file=sys\.stderr` stops at the first `)` inside the f-string. Fix BOTH the regex blind spot and the offending line.
2. **CCONF-05 / CFED-03 latent gap** — `schema_version` is stamped only at `to_json()` time (`export.py:332`); `build()` and `build_from_json()` never write `G.graph['schema_version']`. Stamp at construction so in-memory consumers see the contract.
3. **VFIX-02 docs drift** — README, `--help`, and 7 platform skill files document precedence as `--output > profile > --obsidian-dir > legacy default`, but the runtime chain is `--output > profile > option-b > --obsidian-dir > legacy default`. Functional behavior is correct; docs are wrong. Correct the wording.

**Out of scope:** Filling missing/partial Nyquist VALIDATION.md gaps for phases 63, 64, 66, 67, 70, 70.1 — those are tracked separately as `/gsd-validate-phase` tasks per the audit. This phase is NOT a full-Nyquist phase.

</domain>

<decisions>
## Implementation Decisions

### AUDIT-02 stderr sweep
- **D-01:** Fix BOTH the regex blind spot AND rewrite `build.py:251`. Tighten `tests/test_stderr_contract.py:108` so it spans across f-string parens (e.g., extend the grep regex to match across balanced parens, or use a Python AST scan instead of grep). The new test must catch a `print(f"[graphify] Extraction warning ({n}): ...", file=sys.stderr)`-style line.
- **D-02:** Rewrite `build.py:251` to use the `[graphify] hint:` D-04 prefix. Rationale: it's a non-fatal advisory about validation issues that don't block graph construction — `error:` overstates, `info:` understates. Suggested format: `[graphify] hint: extraction has {n} issues, first: {first_error}`.
- **D-03:** Order of operations is implementation choice for the planner — could land as one commit (regex + line together) or two (TDD: tighten regex → watch fail → fix line). Either is acceptable; planner picks based on atomicity.

### CCONF-05 / CFED-03 schema_version stamp
- **D-04:** Stamp `G.graph['schema_version'] = SCHEMA_VERSION` in BOTH `build.py::build()` and `build.py::build_from_json()`. Federation (`federate.py`) inherits naturally because it builds via `build_from_json(peers=...)`.
- **D-05:** Promote the hardcoded `"1.13"` in `export.py:332` to a named constant `SCHEMA_VERSION` in `graphify/build.py`. `export.py::to_json` imports it from `build`. Single source of truth, co-located with the constructors that stamp it.
- **D-06:** Add an invariant test asserting both constructors stamp the version (e.g., `tests/test_build.py::test_schema_version_stamped_in_memory`), and update/extend the existing `to_json` schema test if there is one to read from the constant.

### VFIX-02 docs precedence
- **D-07:** Define ONE canonical precedence-chain string and propagate it verbatim to all 9 locations (README.md:417, graphify/__main__.py --help around lines 1746 & 1767, and 7 skill files: skill.md, skill-codex.md, skill-claw.md, skill-droid.md, skill-opencode.md, skill-trae.md, skill-windows.md). Canonical form: `--output > profile > option-b (vault) > --obsidian-dir > legacy default (graphify-out/obsidian)`. Add a one-line gloss explaining "option-b (vault)" the first time it appears in each long-form doc.
- **D-08:** Source-of-truth comment for the chain is `__main__.py:2021` (`# Precedence (D-08): --output > profile > option-b (vault) > --obsidian-dir > legacy default`) — already correct. The `--help` strings at __main__.py:1746 and :1767 currently put option-b AFTER --obsidian-dir; those need fixing too (the audit mentions only README + 7 skill files but --help is also wrong).
- **D-09:** No shared-constant refactor for the docs string (Markdown / argparse help strings can't easily import). Verbatim duplication across 9 sites + an invariant grep test is good enough.

### Test gate (targeted, not full Nyquist)
- **D-10:** Three new/updated tests, no VALIDATION.md / Nyquist wave:
  1. Regex regression test in `tests/test_stderr_contract.py` — assert the tightened scanner catches a representative f-string-with-paren violation (synthetic fixture or a parametrized case).
  2. Schema-version invariant test in `tests/test_build.py` — assert `build()` and `build_from_json()` both populate `G.graph['schema_version']` and that `export.to_json()` reads the same constant.
  3. Docs-precedence invariant test (new file or extension of existing test) — grep all 9 doc sites for the OLD wrong wording (`profile > --obsidian-dir > legacy`) and fail if any match; optionally also grep for the NEW canonical string and assert presence.

### Claude's Discretion
- Exact regex vs AST choice for the stderr scanner (D-01) — planner/researcher decides based on robustness vs. complexity. AST is more correct but heavier; a balanced-paren regex may suffice.
- Whether to land as 3 separate commits (one per item) or one bundled commit — let the planner decide, but per repo convention atomic-per-item is safer.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `graphify/build.py::build()` and `build_from_json()` — single-attention-point modules; both create the `nx.DiGraph()`/`nx.Graph()` in a single statement. Easy to add `G.graph['schema_version'] = SCHEMA_VERSION` immediately after construction.
- `tests/test_stderr_contract.py::test_no_outlier_stderr_prefixes_in_source` — already greps `graphify/` recursively; just needs a tighter regex (or AST replacement) at line 108.
- `tests/test_stderr_contract.py::test_strict_prefix_whitelist` — fixture-based whitelist test already validates the three prefixes (`error:`, `info:`, `hint:`); aligning build.py:251 to `hint:` requires no fixture change beyond confirming the new line shape passes.

### Established Patterns
- D-04 strict three-prefix whitelist (`[graphify] error:` / `info:` / `hint:`) is already documented and enforced — this phase doesn't introduce new contract surface, only closes the enforcement gap.
- Module-level constants live alongside their primary consumers (e.g., `_MAX_FETCH_BYTES` in `security.py`). `SCHEMA_VERSION` in `build.py` follows that pattern.
- Per CLAUDE.md, validation returns lists of error strings (not raises); `build.py:251`'s warning print is the user-facing surface for those errors. Keep that semantics.

### Integration Points
- `export.py::to_json` (line 332) currently reads `getattr(G, "graph", {}).get("schema_version", "1.13")` — once constructors stamp the version, the fallback can either remain (defensive) or be replaced by `SCHEMA_VERSION` import (purer). Planner decides.
- `federate.py::federate` and `build_from_json(peers=...)` — verify the stamp survives federation; should be automatic since federation rebuilds via `build_from_json`.

</code_context>
