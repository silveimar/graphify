<!-- update-vault sidecar auto-discovery (verified at graphify/migration.py:166-265, graphify/elicit.py:294-300) -->

`run_update_vault` calls `merge_elicitation_into_build_inputs([extraction], artifacts_dir)`. If `<artifacts_dir>/elicitation.json` exists, its nodes/edges are merged into the build inputs BEFORE clustering. If absent, the call is a silent no-op. There is no CLI flag to opt in/out.

<!-- Critical artifacts_dir alignment (RESEARCH Pitfall 3) -->

Without `--vault` on the elicit call, `artifacts_dir` resolves to `<cwd>/graphify-out` — which differs from the update-vault `<vault>.parent/graphify-out`. The two subprocess calls would write/read different directories → silent merge no-op → assertions fail with no obvious cause.

THE TEST MUST PASS `--vault <vault>` TO THE ELICIT CALL.

<!-- Two-step preview→apply for update-vault (verified at graphify/migration.py:218-265) -->

Same constraint as Plan 01: `update-vault` without `--apply` is preview-only; notes are not written to disk. To assert against rendered note bodies, the test must run preview then `--apply --plan-id <id>`.
