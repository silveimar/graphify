# Phase 59.1 Plan 03: --version flag wiring + doctor extension Summary

Multi-line `graphify --version` / `-V` output now emits package version, per-platform skill stamps with `~`-shortened install paths, Python interpreter version, and editable-vs-site-packages install mode; `graphify doctor` prepends a `version sync:` section showing per-platform stamp/package/status, with shared `<corrupt>` rendering for oversized/unreadable stamps.

## What Shipped

- **`_read_stamp_or_corrupt(version_file)`** — shared stamp reader with the same 1024-byte size guard as Plan 02's `_check_skill_version`. Returns `None` (missing), `"<corrupt>"` (oversized/decode error), or stripped stamp string. Never raises.
- **`_tilde(p, home)`** — renders absolute paths as `~/relative` when under `home`.
- **`_render_version_block(home=None)`** — pure renderer producing the four-section block: `graphify <pkg>`, `skill stamps:` (one row per `_PLATFORM_CONFIG` entry), `python:`, `install:` (editable / site-packages detected via `Path(graphify.__file__).resolve()`).
- **`_render_doctor_version_sync(home=None)`** — pure renderer producing `version sync:` followed by `  <name>: stamp=<v> package=<v> status=<...>` rows. Statuses: `✓ in sync`, `! drifted-newer`, `! drifted-older`, `— not installed`. `<corrupt>` stamps render with `! drifted-newer` status.
- **`--version` / `-V` handler** at `__main__.py:1335` rewired to `print(_render_version_block())`.
- **Doctor handler** at `__main__.py:2841` (`elif cmd == "doctor":`) now calls `print(_render_doctor_version_sync()); print()` as the FIRST two statements; existing argparse + dispatch logic untouched.

## Threat Mitigations Verified

- **T-59.1.03-02 (oversized/unreadable stamp tampering):** Both `_render_version_block` and `_render_doctor_version_sync` go through `_read_stamp_or_corrupt`, which applies the 1024-byte size guard, wraps `read_text` in `try/except (OSError, UnicodeDecodeError)`, and returns the literal string `<corrupt>` instead of raising. Two dedicated tests (`test_render_version_block_handles_corrupt_stamp`, `test_doctor_version_sync_renders_corrupt_stamp`) enforce this contract.
- **T-59.1.03-04 (side-effects on read-only command):** `--version` skip list unchanged; `_render_version_block` contains no `write_text` calls. Test `test_version_flag_does_not_write_stamp` asserts both bytes and mtime unchanged after invoking `main()` with `--version`.
