# Phase 59.1 Plan 02: TDD core — silent auto-self-heal for skill stamp drift Summary

Rewrites `_check_skill_version()` in `graphify/__main__.py` so that stamp-older drift is silently auto-healed by rewriting `.graphify_version` to the running `__version__`, eliminating the noisy `skill stamp ('0.4.7') is older than the installed package; run 'graphify install' to update the skill. (package is '1.0.0')` warning that fired on every CLI invocation after `pip install -e .` upgrades. Uses TDD RED/GREEN gating with a 1024-byte size guard at the read entry point and a D-05 silent-abort guard when `importlib.metadata` cannot resolve the package version.

## Behavior Implemented

`graphify/__main__.py::_check_skill_version(skill_dst: Path) -> None` now executes the following steps in order:

1. Compute `version_file = skill_dst.parent / ".graphify_version"`. If it does not exist, return (D-02 preserved — no-op).
2. **Size guard (T-59.1.02-02):** if `version_file.stat().st_size > 1024`, return silently (treat as corrupt; bounds the read).
3. Read `installed`, compute `pkg = __version__`.
4. **D-05 guard:** if `not pkg`, return silently — never write the stamp.
5. If `installed == pkg`, return.
6. Compute numeric-prefix tuples `it_pad`, `pt_pad` (existing helper preserved).
7. **Heal path (D-01):** if `it_pad < pt_pad`, `try: version_file.write_text(pkg, encoding="utf-8"); return`. On `OSError`/`PermissionError`, **fall through (D-04)** to the legacy two-line stderr warning.
8. **D-03 preservation:** if `it_pad > pt_pad`, emit the legacy `"newer than the installed package..."` warning; stamp NOT rewritten.
9. Otherwise (equal numeric prefix, unequal strings) emit the legacy `"differs from"` warning.

The two-line warning shape is identical to the previous implementation (no new disclosure surface, T-59.1.02-04).
