7. Walk the last `max_snapshots` snapshots with memory discipline (Pattern E):
   ```python
   # {node_id: [(timestamp, community, degree), ...]}
   history: dict[str, list[tuple]] = {}
   for path in snaps[-max_snapshots:]:
       G_snap, _, meta_snap = load_snapshot(path)
       ts = meta_snap.get("timestamp", path.stem)
       for nid, d in G_snap.nodes(data=True):
           history.setdefault(nid, []).append((ts, d.get("community", -1), G_snap.degree(nid)))
       del G_snap   # CRITICAL
   # Add current live G as the tip:
   for nid, d in G.nodes(data=True):
       history.setdefault(nid, []).append(("current", d.get("community", -1), G.degree(nid)))
   ```
8. Compute per-node trend score:
   - `community_changes = len(set(c for _, c, _ in entries)) - 1` (0 = never changed, >0 = drifted)
   - `degree_delta = entries[-1][2] - entries[0][2]` (signed centrality change)
   - `trend_score = community_changes * 2 + abs(degree_delta)` (simple weighted sum; planner's discretion per D-18 "no new algorithms" — this is composition, not a new algorithm)
9. Filter to nodes present in at least 2 snapshots (ignores nodes that only appeared once — noise). Sort by `trend_score` descending. Take top_n.
10. Build `text_body`: Layer-1 narrative — top drifting nodes with label, community change count, degree delta. Clamp to `budget * 3`.
11. Build `meta`: `{"status": "ok", "layer": 1, "search_strategy": "drift", "cardinality_estimate": None, "continuation_token": None, "snapshot_count": min(len(snaps), max_snapshots), "drift_count": len(top_drifters), "nodes_scanned": len(history)}`.
12. Return `text_body + QUERY_GRAPH_META_SENTINEL + json.dumps(meta, ensure_ascii=False)`.

6. Compute current communities: `comms_current = _communities_from_graph(G)` (already in scope as `communities` in serve()). But we need community membership by ID to detect NEW communities. A "new cluster" is a current community whose MEMBER-SET has no overlap with any prior community — more robust than integer-id equality since Leiden re-numbers across runs.
7. For each current community id `c` with members `M_c`:
   - If `M_c` ∩ `prev_node_ids == empty` → all members are brand-new → "newly formed cluster of entirely new nodes"
   - Else for each prior community `p` with member set `P_p`: if Jaccard overlap `|M_c ∩ P_p| / |M_c ∪ P_p|` ≥ 0.3 → treat as "descendant of p" (not new). Else → new.
   - The simpler rule (use THIS rule per D-18 — do NOT invent a clustering algorithm): a community is "new" if NONE of its members appeared in any prior community. This is a pure set operation over data already produced by prior pipeline runs.
   Define rule as: `is_new = (M_c & prev_node_ids) == empty_set` OR `(best_jaccard(M_c, prior_communities) < 0.3)`. Pick the simpler rule — **`is_new = not any(set(M_c) & set(P_p) for P_p in comms_prev.values())`** — reject communities that overlap with any prior. Document the choice in code comment.
8. If `len(new_clusters) == 0` → status="no_change" + text_body="No new clusters formed since the last run. The graph structure is stable." + meta with SENTINEL.
9. Else build `text_body`: one line per new cluster with community id, size, 2–3 representative labels (pick nodes with highest degree). Clamp to `budget * 3`.
10. Build `meta`: `{"status": "ok", "layer": 1, "search_strategy": "emerge", "cardinality_estimate": None, "continuation_token": None, "snapshot_count": len(snaps), "new_cluster_count": len(new_clusters), "new_cluster_ids": [c for c, _ in new_clusters]}`.
11. Return envelope.
