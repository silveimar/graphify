## What Was Built

A single new test module, `tests/test_output_path_matrix.py`, containing seven `def test_` functions:

4. `test_cwd_parent_no_pin_no_cli_returns_default_no_vault` — locks D-12 backcompat: parent cwd with no vault detection returns default no-vault paths.

6. `test_no_doubled_vault_name_segment_in_any_resolution` — parametrized regression sentinel across resolutions 1, 2, 3, 5.
7. `test_obsidian_dispatch_parent_cwd_with_obsidian_dir_no_doubled_segment` — **the dedicated RED sentinel.** Drives a minimal NetworkX graph through `to_obsidian(dry_run=True)` with a relative `obsidian_dir` arg (exactly what `__main__.py:1869` forwards in the user_passed_obsidian_dir branch) and asserts no `MergePlan.actions` path contains the vault directory name twice. **Currently FAILS with `AssertionError: doubled 'uat70-vault' segment in planned path .../uat70-vault/uat70-vault/Atlas/Sources/Graphify/Things/...md`** — exactly the UAT70 reproducer.

## Deviations from Plan

### [Rule 3 - Blocking issue] Used monkeypatch.chdir in Test 7

- **Found during:** Task 1 implementation
- **Issue:** The plan's `<action>` block instructs "do NOT use `monkeypatch.chdir`". However, Test 7 must reproduce the dispatch bug, which requires the relative `--obsidian-dir` arg to resolve against the vault's parent (not the pytest process cwd, which is the repo root). Without confining cwd, `to_obsidian` would create directories inside the graphify repo, violating CLAUDE.md's "no fs side-effects outside tmp_path" rule.
- **Fix:** Test 7 calls `monkeypatch.chdir(tmp_path)` once, immediately after fixture setup. All other tests honor the plan's instruction and pass paths explicitly without cwd manipulation.
- **Rationale:** The tmp_path confinement rule (CLAUDE.md, project-wide) takes precedence over the plan's per-test instruction. The two prohibitions conflict for this specific reproduction; chdir-to-tmp_path is the only way to satisfy both the bug-reproduction goal and the fs-isolation contract.
- **Files modified:** tests/test_output_path_matrix.py (Test 7 only)
- **Commit:** d64e892

### Resolver-only invariants pass today

- **Found during:** Iterative test development
- **Observation:** Tests 1–6 (resolver invariants on graphify.output.resolve_output and resolve_execution_paths) all PASS today. The plan's `<must_haves>` predicted "≥1 sentinel test fails today (the doubled-segment regression scenarios)" — but at the public resolver API layer, the production code is already correct. The actual bug lives downstream in `__main__.py` dispatch (specifically the user_passed_obsidian_dir branch around line 1869) and surfaces only when a *relative* `--obsidian-dir` argument is forwarded into `to_obsidian` via `Path(...).resolve()` against process cwd.
- **Action:** Test 7 was rewritten as an end-to-end driver that exercises the actual buggy dispatch shape via `to_obsidian(dry_run=True)`. This produces a real RED failure that maps 1:1 to the UAT bug report. Plan 70.1-02 will fix the dispatch and turn Test 7 GREEN without modifying Tests 1–6.
