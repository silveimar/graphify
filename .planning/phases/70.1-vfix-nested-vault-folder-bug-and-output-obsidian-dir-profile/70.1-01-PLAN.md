  <behavior>
    Seven pure-unit tests, all using tmp_path. Each builds a minimal vault fixture with
    `.obsidian/` + `.graphify/profile.yaml` containing:
      output:
        mode: vault-relative
        path: .
      taxonomy:
        root: Atlas/Sources/Graphify
      folder_mapping:
        thing: Atlas/Sources/Graphify/Things/

    The canonical expected `notes_dir` is `vault.resolve()` (the vault root itself, since
    `output.path == "."`). The taxonomy/folder_mapping join happens later in to_obsidian
    and is NOT under test here. Tests assert `resolved.notes_dir == vault.resolve()` and
    that `resolved.notes_dir` does NOT contain a doubled `vault.name` segment (regression
    sentinel for the nested-folder bug).

    - Test 1 — `test_cwd_in_vault_profile_dot_resolves_to_vault_root`:
        `resolve_output(vault)` → `notes_dir == vault.resolve()`, `source == "profile"`,
        `vault_detected is True`. Asserts no path component equals `vault.name` twice.

    - Test 2 — `test_cwd_in_vault_with_explicit_vault_pin_resolves_identically`:
        `resolve_execution_paths(vault, explicit_vault=vault)` → notes_dir identical to Test 1.

    - Test 3 — `test_cwd_parent_with_explicit_vault_pin_resolves_identically`:
        `resolve_execution_paths(parent_dir, explicit_vault=vault)` → notes_dir identical
        to Test 1 (key invariant: parent-cwd must NOT prepend itself onto the vault path).

    - Test 4 — `test_cwd_parent_no_pin_no_cli_returns_default_no_vault`:
        `resolve_output(parent_dir)` (parent has no `.obsidian/`) → `vault_detected is False`,
        `source == "default"`, `notes_dir == Path("graphify-out/obsidian")` (D-12 backcompat).
        This locks the "do not silently scan parent for vault" behavior.

    - Test 5 — `test_cli_output_overrides_profile_path_dot`:
        With profile present and `cli_output=str(tmp_path/"custom-out")`,
        `resolve_output(vault, cli_output=...)` → `source == "cli-flag"`,
        `notes_dir == (tmp_path/"custom-out").resolve()`. D-08 precedence holds.

    - Test 6 — `test_no_doubled_vault_name_segment_in_any_resolution`:
        Parametrized over the four resolutions above (Tests 1, 2, 3, 5). For each result,
        assert `[p for p in resolved.notes_dir.parts if p == vault.name]` has length ≤ 1.
        This is the dedicated regression sentinel for the UAT nested-folder bug.

    - Test 7 — `test_obsidian_dispatch_parent_cwd_with_obsidian_dir_no_doubled_segment`:
        Locks the `--obsidian` dispatch precedence path. From `parent_dir` (no `.obsidian/`
        at cwd, no `--vault` pin), simulate the dispatch by calling
        `resolve_execution_paths(parent_dir)` to get the no-vault baseline, then
        independently resolve `Path("uat70-vault").resolve()` against `parent_dir` (the
        `user_passed_obsidian_dir` branch in __main__.py). The resulting absolute
        obsidian_dir, when joined with the vault's profile output (`.`), MUST equal
        `vault.resolve()` exactly — no doubled `vault.name` segment. This test belongs
        in the matrix file (not a separate file) so the RED → GREEN cycle stays coherent
        and Plan 70.1-02 Task 2 does not modify the test file mid-cycle.

    All tests must capture stderr via `capsys` where the resolver emits detection lines
    (Tests 1, 2, 3 emit a "vault detected" stderr line; tolerate but do not assert on
    exact text — that is covered by tests/test_output.py).
