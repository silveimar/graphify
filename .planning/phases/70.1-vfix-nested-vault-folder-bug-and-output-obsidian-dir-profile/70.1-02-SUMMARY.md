## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Root-cause fix lives in `graphify/export.py`, not `graphify/output.py`**

- **Found during:** Task 1 diagnosis (running `pytest tests/test_output_path_matrix.py -q`)
- **Issue:** Plan Task 1 hypothesized the bug might be in `resolve_output` / `resolve_execution_paths`. Investigation showed matrix tests 1-6 already pass — those resolvers are correct. The doubled-segment bug is in `to_obsidian`: when `output_dir` is relative, `out = Path(output_dir)` stays relative; `target_path = out / folder / filename` therefore contains `vault.name`; downstream `_validate_target(target_path, vault_dir)` joins that relative target against an already-resolved `vault_dir` ending in the same `vault.name` — producing `<vault>/<vault>/Atlas/...` .
- **Fix:** Resolve `output_dir` to absolute at the top of `to_obsidian` so every `out / ...` path is absolute and `_validate_target` strips the vault prefix cleanly.
- **Files modified:** `graphify/export.py` (+11 lines, no behavior change for absolute output_dir callers)
- **Commit:** `37ddc3f`
