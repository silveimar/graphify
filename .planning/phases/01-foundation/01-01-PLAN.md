<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vault filesystem -> profile.py | Untrusted YAML from user vault (.graphify/profile.yaml) |
| Profile folder_mapping values -> filesystem writes | Profile-derived paths could escape vault directory |
| Node labels -> YAML frontmatter | Untrusted graph data injected into YAML frontmatter values |
| Community names -> Obsidian tags | Untrusted names become tag paths |
| Node labels -> filenames | Untrusted labels become filesystem paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | validate_vault_path() | mitigate | Path.resolve() + relative_to() check prevents ../traversal and symlink escape |
| T-01-02 | Injection | safe_frontmatter_value() | mitigate | Quote-wrap values containing YAML special chars (: # [ ] { }), escape internal quotes, strip newlines |
| T-01-03 | Injection | safe_tag() | mitigate | Slugify to [a-z0-9-] only, strip all special chars including / and + |
| T-01-04 | Injection | safe_filename() | mitigate | Strip OS-illegal chars via regex, NFC normalize, length cap with hash suffix |
| T-01-05 | Tampering | load_profile() / yaml.safe_load | mitigate | Use yaml.safe_load (not yaml.load) to prevent arbitrary object instantiation from vault YAML |
| T-01-06 | Denial of Service | safe_filename() length cap | mitigate | Cap at 200 chars to prevent OS path length errors |
| T-01-07 | Information Disclosure | load_profile() error messages | accept | Error messages reveal profile path — acceptable since user controls their own vault |
</threat_model>
