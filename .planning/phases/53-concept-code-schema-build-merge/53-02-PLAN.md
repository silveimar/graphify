## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted extraction JSON → validate_extraction | Schema gate; must reject malformed input before build/graph phases |
| `evidence` field value → error message | Embeds user-controlled string in stderr/error list |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53.02-01 | Spoofing | `evidence` field accepting arbitrary string | mitigate | Whitelist via `KNOWN_EVIDENCE_VALUES` frozenset; reject anything else with explicit error. |
| T-53.02-02 | Tampering | Bypassing the rule by spelling `relation` differently (case/whitespace) | mitigate | `frozenset` lookup is exact-match; no case folding. Existing `warn_unknown_relations` already flags malformed relations as a secondary signal. |
| T-53.02-03 | Information Disclosure | `evidence` value embedded in error message via `repr()` | accept | Error strings flow to stderr via `[graphify]` prefix; no log persistence. `repr()` quotes control chars. Consistent with existing `confidence`-enum error pattern. |
| T-53.02-04 | Denial of Service | Large `evidence` string causing huge error message | accept | Validation runs once per edge; no amplification. Caller is responsible for input size (CCODE-05 patterns in `security.py` cap upstream). |
| T-53.02-05 | Elevation of Privilege | Coercing `confidence_score` to bypass [0,1] gate | mitigate | `float()` cast inside try/except; bounds check via `0.0 <= score <= 1.0` (handles NaN — `float('nan')` fails the comparison). |
| T-53.02-06 | Repudiation (audit) | Edge accepted without evidence trail | mitigate | `evidence` field IS the audit trail for EXTRACTED claims; INFERRED carries explicit `confidence_score`. AMBIGUOUS is the only escape hatch (per D-53.09, intentional). |
</threat_model>
