# Phase 9.1 Plan 01: Telemetry Sidecar and Derived Edge Detection Summary

Query telemetry infrastructure added to MCP server: 6 new functions in serve.py tracking edge traversal counts, computing logarithmic weights, decaying stale counters, and auto-proposing 2-hop shortcut edges when traversal threshold is met.

## What Was Built

### Telemetry Sidecar (Task 1)
- `_load_telemetry(path)` / `_save_telemetry(out_dir, data)` -- atomic JSON persistence following _save_agent_edges pattern
- `_record_traversal(telemetry, edges)` -- normalizes edge keys as `min(u,v):max(u,v)` for undirected graph consistency
- `_edge_weight(traversals)` -- `1.0 + log(t)` clamped to [1.0, 10.0]
- `_decay_telemetry(telemetry, multiplier)` -- multiplies counters, removes zero entries (ready for Plan 03 wiring)
- Wired into `serve()` startup (loads telemetry.json) and `_tool_query_graph()` (records + saves after each BFS/DFS)

### Derived Edge Detection (Task 2)
- `_check_derived_edges(G, telemetry, out_dir, agent_edges)` -- scans 2-hop neighbor pairs where both segments meet traversal threshold
- Adds `derived_shortcut` edges to agent-edges.json with INFERRED confidence (0.7 score)
- Prevents duplicates across repeated calls and skips existing graph edges
- Wired into `_tool_query_graph()` after telemetry recording
