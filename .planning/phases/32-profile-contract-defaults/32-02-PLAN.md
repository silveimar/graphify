<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.graphify/profile.yaml` -> profile validator | User-authored profile data controls output folders and mapping thresholds. |
| `.graphify/templates/community.md` -> preflight warnings | User-authored template files can request deprecated output classes. |
| default profile -> export/mapping runtime | Built-in defaults route generated notes when no vault profile exists. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-04 | Tampering | `taxonomy.folders.*` | mitigate | Reject `..`, absolute paths, `~`, non-string entries, and unsupported folder names in `validate_profile()`. |
| T-32-05 | Tampering | `taxonomy` schema | mitigate | Treat unsupported top-level taxonomy keys as validation errors to prevent silent misrouting. |
| T-32-06 | Repudiation | `mapping.moc_threshold` | mitigate | Return a hard validation error naming the deprecated key and the replacement `mapping.min_community_size`. |
| T-32-07 | Repudiation | community overview templates | mitigate | Emit preflight warnings naming `community.md` or the explicit community template setting and pointing to MOC-only output/migration guidance. |
| T-32-08 | Denial of Service | `mapping.min_community_size` | mitigate | Reject bool and non-int values; reject values below 1 so routing cannot be disabled through malformed config. |
</threat_model>
