<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| profile preflight -> doctor report | Shared validator findings become user-facing diagnostic output. |
| doctor report -> CLI exit code | Errors must block while warnings guide migration without fatal failure. |
| deprecated profile/template use -> user migration behavior | Ambiguous messages can cause users to preserve hard-deprecated community overview output. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-13 | Repudiation | `graphify doctor` validation source | mitigate | Replace doctor-local validation with `validate_profile_preflight()` so doctor and `--validate-profile` share one source. |
| T-32-14 | Tampering | unsupported taxonomy keys | mitigate | Surface preflight errors in doctor as blocking profile validation errors. |
| T-32-15 | Repudiation | deprecated community overview output | mitigate | Render explicit warning and fix guidance naming MOC-only output and migration guidance. |
| T-32-16 | Denial of Service | doctor warnings | mitigate | Preserve `is_misconfigured()` semantics so warnings alone do not block doctor success. |
</threat_model>
