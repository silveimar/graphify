<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> local filesystem | `--input`, `--vault`, `--plan-id`, and optional repo identity are untrusted strings controlling reads, writes, and artifact lookup. |
| Current preview -> stored preview artifact | Apply compares a fresh run to a previously written artifact that may be stale or edited. |
| Existing vault metadata -> merge classification | Manifest/frontmatter repo identity can cause SKIP_CONFLICT and must not be trusted blindly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-11 | Tampering | `update-vault --input/--vault` | mitigate | Resolve both paths, require input existence, require target `.obsidian/`, and use `resolve_output(vault_dir)` plus `validate_vault_path()` downstream for vault-relative writes. |
| T-35-12 | Tampering | `--plan-id` apply lookup | mitigate | Use Plan 01 `load_migration_plan()` validation so path separators, `..`, and malformed IDs cannot select arbitrary files. |
| T-35-13 | Tampering | stale, mismatched, or reclassified preview apply | mitigate | Rerun current dry-run preview, compare deterministic content-digest `plan_id`, input path, vault path, and repo identity, then apply only the validated classified CREATE/UPDATE/REPLACE actions through `apply_merge_plan()` so repo-drift SKIP_CONFLICT rows cannot be bypassed. |
| T-35-14 | Spoofing | repo identity injection | mitigate | Use `resolve_repo_identity()` and sanitized CODE metadata from Plan 02; never trust raw CLI/profile values in tags or frontmatter. |
| T-35-15 | Tampering | silent overwrite of user-modified notes | mitigate | Keep writes delegated to `to_obsidian(..., dry_run=False)` and `apply_merge_plan()`, preserving SKIP_PRESERVE/user-modified checks from `graphify.merge`. |
| T-35-16 | Repudiation | legacy cleanup data loss | mitigate | Keep `_COMMUNITY_*` files as ORPHAN/review rows only; tests assert apply leaves them present and no CLI branch calls deletion/move APIs for legacy notes. |
</threat_model>
