    **5. Step A4 — Compute Borda scores and assemble results (bash block):**
    After all judge responses are collected for a lens, compute Borda scores inline:

    ```bash
    $(cat graphify-out/.graphify_python) -c "
    import json, random
    from pathlib import Path

    # Input: judge_rankings is a list of lists, e.g. [['A','B','AB'], ['A','AB','B'], ['AB','A','B']]
    # (parsed and unshuffled by the skill orchestrator above)
    judge_rankings = JUDGE_RANKINGS_PLACEHOLDER

    # Borda count: n=3 candidates, 1st=2pts, 2nd=1pt, 3rd=0pts
    scores = {'A': 0, 'B': 0, 'AB': 0}
    for ranking in judge_rankings:
        for rank, candidate in enumerate(ranking):
            scores[candidate] += (2 - rank)

    winner = max(scores, key=scores.get)
    total_points = sum(scores.values())
    confidence = scores[winner] / total_points if total_points > 0 else 0.0

    # Confidence label
    if confidence >= 0.8:
        conf_label = 'high'
    elif confidence >= 0.5:
        conf_label = 'medium'
    else:
        conf_label = 'low'

    print(json.dumps({'winner': winner, 'scores': scores, 'confidence': confidence, 'confidence_label': conf_label}))
    "
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM output in tournament rounds | Each round produces untrusted text that flows to next round and to render_analysis() |
| Judge ranking parsing | LLM judge output must be parsed into structured ranking — could be malformed |
| graph.json to LLM context | Graph data serialized into prompts — node labels could contain prompt injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | T (Tampering) | Judge ranking parse | mitigate | Validate judge output matches expected format "1st: Analysis-N 2nd: Analysis-N 3rd: Analysis-N" before processing. If malformed, skip that judge (degrade gracefully with 2 judges instead of 3). |
| T-09-05 | S (Spoofing) | Context leakage between rounds | mitigate | Each tournament round is a separate LLM call. Only TEXT output flows between rounds, never system prompts or reasoning traces. Anti-pattern comments enforce this. |
| T-09-06 | T (Tampering) | Prompt injection via node labels | accept | Node labels come from graphify's own extraction pipeline which already sanitizes via security.py. The graph context is a compact summary, not raw user input. Low risk. |
| T-09-07 | D (Denial of Service) | Token cost of 4 lenses x 6 calls | accept | Tournament is opt-in (/graphify analyze). User explicitly requests it. Report token estimate before running. |
</threat_model>
