<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| graph/community data -> concept signature | Node IDs, labels, and source files are converted into cache keys and provenance metadata. |
| LLM candidate -> concept title | Untrusted model output can affect generated MOC labels and downstream markdown sinks. |
| naming cache sidecar -> future run | Prior generated state can be reused across runs and must not poison unrelated communities. |
| artifacts_dir -> cache path | Generated sidecar path must remain under graphify-out-style artifact directories. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-11 | Tampering | `_validate_title_candidate()` | mitigate | Reject path-like, wikilink-breaking, template-breaking, control-character, empty, duplicate, overly generic, and too-long titles before cache persistence. |
| T-33-12 | Tampering | concept cache keys | mitigate | Key exact reuse by SHA-256 community signature from sorted member IDs, labels, and source files; do not key by raw community ID alone. |
| T-33-13 | Repudiation | concept provenance | mitigate | Store source, signature, reason, and top terms for every resolved name so users can inspect why a title was chosen. |
| T-33-14 | Denial of Service | LLM title length | mitigate | Reject or cap long candidates before persistence; still pass accepted/fallback labels through existing sink-specific sanitizers later. |
| T-33-15 | Tampering | tolerant cache reuse | mitigate | Permit tolerant reuse only for strong top-term overlap and mark provenance as `cache-tolerant`; fallback when overlap is weak. |
| T-33-16 | Information Disclosure | concept sidecar | accept | Sidecar stores graph-derived labels/source_file metadata already present in generated graph outputs; no secrets should be introduced by this helper. |
</threat_model>
