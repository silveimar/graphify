<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Loaded migration artifact -> filesystem | JSON rows may be stale or tampered and must be revalidated before archive movement. |
| Target vault -> archive directory | User vault paths cross into `graphify-out/migrations/archive/` and must remain confined to both roots. |
| Apply writes -> archive movement | A failed write must not trigger movement of legacy user files. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-01 | Tampering | `load_migration_plan()` / `run_update_vault()` | mitigate | Keep plan digest validation and `validate_plan_matches_request()` before archive; tests must prove stale plans leave legacy files untouched. |
| T-36-02 | Denial of Service | `run_update_vault()` apply branch | mitigate | Call archive only after `apply_merge_plan()` has no failures; tests must prove failed apply does not archive. |
| T-36-03 | Tampering / Elevation of Privilege | `archive_legacy_notes()` | mitigate | Validate source with `validate_vault_path()` and validate destination relative to archive root; reject traversal, absolute escape, and duplicate destinations before moving any file. |
| T-36-04 | Repudiation | CLI output / archive metadata | mitigate | Return and print archive metadata including source and archive paths so rollback is auditable. |
</threat_model>
