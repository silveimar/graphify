**Finding:** OBS-01 (`tag:community/` syntax) and OBS-02 (read-merge-write for user settings in graph.json) are BOTH rooted in the `safe_tag()` helper producing the correct slug form. That helper has:
- 5 direct unit tests in tests/test_profile.py (normal, leading-digit, special-chars, empty, slashes-and-plus)
- Transitive coverage in test_mapping.py (community_tag == safe_tag(name))
- Transitive coverage in test_templates.py (rendered MOC dataview query uses `community/<safe_tag>`)

These tests survive the graph.json deletion untouched. The invariant "community tags use `community/<slugified-name>` syntax, never `#<raw>`" is covered by 5 + 3 + 2 = 10 surviving tests.

**Decision:** Delete `test_graph_json_tag_syntax` and `test_graph_json_preserves_user_settings` (the graph.json feature is removed from to_obsidian per D-74).
