<objective>
Bug fix: `detect()` in `graphify/detect.py` walks into prior `graphify-out/` output directories
and re-ingests graphify's own exported notes as fresh `document` inputs. This causes nested
`graphify-out/obsidian/graphify-out/obsidian/...` paths on every re-run when graphify is invoked
from a directory containing prior output (e.g. an Obsidian vault root after an --obsidian export).

Purpose: Make the default scan safe — users should never need to opt in to excluding graphify's
own output directory.

Output:
  - `graphify/detect.py` patched with a default ignore for `graphify-out/` (preserving the
    existing `graphify-out/memory/` explicit allow-list)
  - `tests/test_detect.py` regression test asserting `tmp_path/graphify-out/obsidian/foo.md`
    is NOT returned by `detect(tmp_path)`
</objective>
