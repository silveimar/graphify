## Soft Findings

**graphify_managed frontmatter key never emitted.** `render_note` / `render_moc` do not add `graphify_managed: true` to the frontmatter they produce, but `merge._has_fingerprint` (merge.py:443) checks for it as part of D-62's dual-signal ownership fingerprint. Test fixtures under `tests/fixtures/vaults/pristine_graphify/` hand-author the key, creating a shape mismatch with real render output. If a user deletes all sentinel blocks from a graphify-authored file, the next run will misclassify it as `unmanaged_file` and refuse to touch it. Phase 5 must decide whether to (a) add `graphify_managed: true` in `_build_frontmatter_fields`, or (b) declare sentinel-only fingerprinting sufficient and remove `graphify_managed` from fixtures. Affects MRG-01, MRG-02.
