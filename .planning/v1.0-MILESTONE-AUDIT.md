## Tech Debt Summary (non-blocking)

### 1. SUMMARY.md frontmatter schema drift (severity: schema-drift)

Phases use inconsistent field names for requirements:

| Phase | Field name used | Populated? |
|---|---|---|
| 01 | `requirements-completed:` | yes |
| 02 | *(no requirement field)* | no |
| 03 | mix of `requirements-completed:` (empty) and `requirements:` (populated) | partial |
| 04 | `requirements-completed:` | yes |
| 05 | `requirements_closed:` (05-06 only) | partial |

**Impact:** The 3-source cross-reference had to reconstruct requirement coverage from VERIFICATION.md tables instead of trusting SUMMARY frontmatter. It worked, but a consistent schema would have been faster. **Remedy:** future housekeeping pass — pick one canonical field (`requirements_completed:` or `requirements:`), back-fill missing entries, document in `templates/summary.md`. Not blocking for v1.0 completion.

### 2. Skill.md `dry_run = False` placeholder (severity: known-limitation, accepted)

`graphify/skill.md:547` has `dry_run = False  # replace with True if --dry-run was passed`. Documented as **D-73** (CLI is utilities-only; skill is the pipeline driver). No runtime mechanism for agent-driven flag substitution. **Remedy:** none — accepted design decision, already flagged in `05-VERIFICATION.md` Anti-Patterns Found table as "Warning (by design)".
